## 문제 구성

각 코드 슬라이드마다 탭 하나. 최소 아래 4개:

### 탭 1: MinHash 슈도코드 (L6)
슬라이드 p30 코드를 그대로 수록 후 핵심 줄을 빈칸으로:
- 초기화 줄 (M[i,c] ← ?)
- 업데이트 조건 (if ? then)
- 업데이트 값 (M[i,c] ← ?)

### 탭 2: SGD for UV (L13)
슬라이드 p26 슈도코드를 그대로 수록 후 빈칸:
- ε 계산식
- u 업데이트 식
- v 업데이트 식

### 탭 3: Spark Quantile Summary (L3, 2025 기출)
기존 2025 기출 6번 문제:
- combine 함수의 rmin, rmax
- compress 함수의 조건
- query 함수의 조건
⚠️ rmin/rmax 정답은 슬라이드와 기출 문제를 바탕으로 직접 유도하여 정확한 값으로 작성할 것 (별도 공식 답안 파일 없음)

### 탭 4: BFR / SON (L9, L5)
슬라이드에서 확인한 내용 기반으로 구성

## UI 구성

### 레이아웃
- 상단 탭: 알고리즘 이름 (MinHash | SGD | Spark | BFR/SON | ...)
- 좌측 패널 (35%): 문제 배경 설명 + 알고리즘 개요
- 우측 패널 (65%): 코드 블록 + 빈칸 input

### 코드 블록
- monospace 폰트
- 빈칸: `<input>` 박스 (너비 280px, 인라인)
- 각 빈칸 옆 [힌트] 버튼 (토글)

### 버튼
- [채점하기]: 빈칸별 정오 표시 (초록/빨강)
- [정답 보기]: 전체 정답 일괄 표시
- [초기화]: 입력값 리셋

### 채점 후 학습 패널
각 빈칸 정답 확인 후 토글로 열리는 설명:
- "이 줄의 역할: ..."
- "왜 이 값인가: ..."
- "이 줄이 틀리면 어떻게 되는가: ..."
