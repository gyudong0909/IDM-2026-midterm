## T/F 문제 생성 기준

### 수량 및 배분
총 80~100문항. 강의 비중에 비례하여 배분:

| 범위 | 문항 수 |
|------|---------|
| L2 MapReduce + L3 Spark | 12~15개 |
| L4~L5 빈발 아이템셋 | 12~15개 |
| L6~L7 유사 아이템 탐색 (LSH) | 12~15개 |
| L8~L9 클러스터링 | 15~18개 |
| L10~L11 차원 축소 | 12~15개 |
| L12~L13 추천 시스템 | 15~18개 |
| L1 Introduction | 3~5개 |

### 문제 품질 기준
1. **기출 스타일 일치**: 실제 기출 T/F와 동일한 문체·난이도로 작성
2. **이해 테스트**: 암기가 아닌 개념 이해를 묻는 문장
3. **함정 유형 다양화** — 기출에서 추출한 패턴 활용:
   - "always / never / only / directly / guaranteed" 키워드로 절대성 함정
   - 유사한 두 알고리즘 혼동 (예: BFR의 DS vs RS, MinHash vs Random Hyperplane)
   - 조건 누락 (예: "k-means++ is always better than k-means")
   - 인과관계 역전 (예: clustroid = mean이라는 오해)
   - 숫자/방향 반전 (예: 첫 번째 PC = 최소 고유값)
4. **해설 품질**: 정답 이유 + 오해하기 쉬운 이유를 한 줄로 설명
5. **L12/L13 특별 강화**: 신규 범위이므로 더 꼼꼼하게 — Pearson, SGD, 정규화, biases 모델, BPR 포함

## STEP 2: React 앱 구현 스펙 (STEP 1 확인 후 작성)

### 기술 스택
- 순수 React + inline CSS
- 수식 해설이 필요한 문항에 한해 KaTeX CDN 사용:
  ```js
  useEffect(() => {
    const link = document.createElement('link');
    link.rel = 'stylesheet';
    link.href = 'https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.css';
    document.head.appendChild(link);
    const script = document.createElement('script');
    script.src = 'https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.js';
    script.onload = () => setKatexLoaded(true);
    document.head.appendChild(script);
  }, []);
  ```

### 채점 규칙 (실제 시험 동일)
- 정답 클릭: +1점
- 오답 클릭: -1점
- [모름] 클릭: 0점

### UI 구성
- 상단 고정 바: 현재 점수 / 진행 문항 수 / 최대 가능 점수
- 카드 중앙: 영문 문장 표시
- 버튼 3개: [TRUE] [FALSE] [모름]
- 답 클릭 후: 카드 플립 애니메이션 → 뒷면에 정오 여부 + 한국어 해설 표시
- [다음 문제] 버튼
- 마지막 화면: 점수 요약 + 정답률 + 틀린 문제 목록 + [다시 풀기] 버튼
- 문제는 매 세션 셔플
