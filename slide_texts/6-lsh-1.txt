Locality-Sensitivity
• Typical hash functions are random uniform mappings.
  • Let ℎ be a hash function, and 𝐷! and 𝐷" be two documents.
  • ℎ 𝐷! and ℎ 𝐷" are independent of their actual similarity.
• What if we have the locality-sensitive property?
  • 𝑃(ℎ 𝐷! = ℎ 𝐷" ) is large if 𝐷! and 𝐷" are similar.
• Hashing can be used for finding similar documents!

Locality-Sensitive Hashing
• Locality-sensitive hashing (LSH):
  • Finds pairs of similar items in a large set without the quadratic cost.
• Basic idea:
  • Throw items into buckets using several different hash functions.
  • Examine only the item pairs that share a bucket for at least one time.
• Upside: Only a small fraction of pairs are examined.
• Downside: False negatives – some pairs never get considered.

Minhashing
• Minhashing creates signatures from the characteristic matrix.
   • Applies several random permutations of the rows.
   • For each permutation, the minhash value of each set (or column) is
     defined as the index of the first row that has 1.
• The result is a signature matrix.
   • Columns are sets, and rows are minhash values.
   • Normally much smaller than the characteristic matrix.

Subtle Point
• People ask whether the minhash value should be:
   • The original number of the row, or
   • The number in the permuted order (as we did in our example).
• Answer: It doesn’t matter.
   • You only need to be consistent.
   • Assure that two columns get the same value if and only if
      • Their first 1’s in the permuted order are in the same row.

Minhashing and Jaccard Similarity
• Surprising property:
   • Probability that minhash values are same = Jaccard similarity of sets.
• Proof:
   • Given columns 𝑆! and 𝑆", three classes of rows exist:
      • Type X: 1 in both columns.
      • Type Y: 1, 0 (or 0, 1) in columns.
      • Type Z: 0 in both columns.
   • If there are 𝑥 rows of Type X and 𝑦 rows of Type Y,
      • sim 𝑆# , 𝑆" = 𝑥/ 𝑥 + 𝑦 .
      • Pr(ℎ 𝑆# = ℎ 𝑆" ) = Pr meeting type X before type Y = 𝑥/ 𝑥 + 𝑦 .

Implementation of Minhashing
• Explicit permutation is infeasible in a large characteristic matrix.
   • Suppose 1 billion rows.
• Idea to approximate permutation:
   • Apply a random hash function ℎ to row indices 𝑟 = 1, … , 𝑛.
   • Sort the rows in order of their hash values ℎ 𝑟 .
   • The result is considered as random permutation.
• Idea to implement:
   • Create a “slot” matrix, and initialize all elements to ∞.
   • Update each element as we encounter smaller values.

Other Ways to Vectorize Documents
• TF-IDF is a popular way to create (a different type of) signatures.
   • Let 𝑀 be the number of unique words in the corpus.
   • TF-IDF creates a length-𝑀 real-valued signature for each document.
• The 𝑖-th element of signature for document 𝑘 is TF@A × IDF@ :
   • TF$% = 𝑓$% / max 𝑓&% where 𝑓&% is the frequency of term 𝑡 in document 𝑘.
                  &
   • IDF$ = log(𝑁/𝑛$ ) where term 𝑖 appears in 𝑛$ of 𝑁 documents.

Different Properties of Signatures
• The minhashed signatures are integer vectors.
  • We measure the similarity by # of the same integer elements.
• The TF-IDF signatures are real-valued vectors.
  • Two document signatures will not share any same elements.
  • We measure the real-valued distance between signatures.
  • Then, we need a different locality-sensitive hash function!

What are Good Signatures?
• Recently, machine learning (ML) is used to create signatures.
  • This topic is called representation learning.
  • These signatures are more commonly called embeddings.
  • All language models like ChatGPT work on embeddings.
• ML is powerful, but LLMs are even more powerful.
