Overview
• Q: How to do clustering if the data is too large to fit in memory?
• CURE: Simplest approach based on hierarchical clustering.
• BFR: Variant of 𝑘-means for large data.
   • More advanced but introduces more assumptions compared to CURE.
• GRGPF: Clever approach for a non-Euclidean space.
   • Summarizes a cluster without its centroid.

CURE (Clustering Using REpresentatives)
• CURE is a 2-pass algorithm for large disk-resident data.
   • Pass 1: Find clusters from a sampled subset.
   • Pass 2: Assign points to the clusters.
• Use a collection of representative points to represent clusters.
• Assume Euclidean space/distance, with 𝑘 (# of clusters) given.




                                   Jaemin Yoo                        6
CURE: Pass 1
1. Pick a random sample of data.
2. Cluster them in main memory using hierarchical clustering.
   • Merge two clusters when they have close pairs of points.
3. Pick representative points from each cluster.
   • For each cluster, pick a sample of points, as dispersed as possible.
   • Move them a fraction of distance, e.g. 20%, toward the centroid ... why?
4. Merge clusters with the closest pair of representatives.

CURE: Pass 2
1. Rescan the whole dataset.
2. Assign each point 𝑝 in the dataset to the “closest cluster.”
   • Find the closest representative point to 𝑝.
   • Assign 𝑝 to the representative’s cluster.

Why to Move 20% Inward?
• Some of the representatives will be on the boundary of clusters.
• We move them 20% towards the centroids.
• Large, dispersed clusters will shrink more than small, dense ones
• As a result, the algorithm favors a small, dense cluster.

BFR (Bradley, Fayyad, and Reina) Algorithm
• Variant of 𝑘-means for large disk-resident data.
• Assume normally-distributed clusters in a Euclidean space.
   • Clusters are axis-aligned ellipses.
   • Standard deviations in different dimensions may vary.
• Goal: Find cluster centroids efficiently (in Pass 1).
   • Point assignment can be done in Pass 2 through whole data.

Main Idea
• Idea: Keep summary statistics of groups of points.
   1. Read points from disk one main-memory-full at a time.
   2. Summarize most points from previous memory loads.
   3. Memory requirement changes from 𝑂 data to 𝑂 clusters .
• Challenge: Not all points can be summarized effectively.
   • Some points will not create a cluster at all.

Three Classes of Points
• Divide all data points into three sets during the algorithm.
• Discard set (= clusters):
   • Points close to a centroid to be summarized and discarded.
• Compressed set (= miniclusters):
   • Groups of points that are close together but not to any centroid.
   • These points are summarized but not assigned to a cluster.
• Retained set:
   • Isolated points waiting to be assigned to a compressed set.

Overview of the BFR Algorithm
1. Initialize 𝑘 clusters/centroids (as in 𝑘-means)
2. for each chunk in a data file
3.   for each point in the chunk
4.      Assign it to a cluster/minicluster if it is sufficiently close to it
5.   Cluster the remaining points, creating new miniclusters
6.   Try to merge existing clusters and miniclusters if they are close

Selection of the Initial Centroids
• The 𝑘 initial centroids can be selected as in 𝑘-means:
• Option 1: Take 𝑘 random points.
• Option 2: Take a small sample, cluster it, and use the centroids.
• Option 3: Take a sample, pick a random point, and 𝑘 − 1 more.
   • Each as far from the previously selected points as possible.

• Finally, what to do with compressed and retained sets?

Assigning a Point to a Cluster
• For each point 𝑝, choose the cluster with the closest centroid.
• Add 𝑝 to the cluster if the distance is less than a threshold.
   • Suppose the threshold = 4.
   • Then, Pr(value being 4 standard deviations from the mean) < 10#$.

When to Merge Two Clusters?
• Compute the variance of the combined subcluster.
• Combine if the combined variance is below some threshold.
• Many alternatives: E.g., considering density.

GRGPF Algorithm
• Does not require a Euclidean space unlike CURE and BFR.
• Represents clusters by well-chosen sample points in memory.
• Organizes clusters hierarchically, as a tree (not covered today).
   • New point is assigned to a cluster by passing it down the tree.

Justification of the Representation
• Clustroid is the point in the cluster with the smallest rowsum.
• Why the 𝑘 points nearest to the clustroid?
   • If the clustroid changes, the new clustroid would be one of them.
   • Point 𝑝 becomes the new clustroid if rowsum 𝑝 < rowsum 𝑐 .
• Why the 𝑘 points farthest from the clustroid?
   • Used to determine whether two clusters are close enough to merge.
