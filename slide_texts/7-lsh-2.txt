Locality-Sensitive Hashing
• The number of document pairs is still large after minhashing.
   • Minhashing does not reduce document pairs to compare.
   • What if we have 1M documents with 1µs for each comparison?
   • 6 days to compute all similarities.
• Locality-sensitive hashing (LSH) reduces # of candidate pairs.
   • Allows us to focus on pairs that are likely to be similar.

From Signatures to Buckets
• General approach: Hash minhashed signatures several times.
  • We know similar documents have more shared signatures.
  • Use only a subset of the signatures for each hash function.
• Any pair hashed to the same bucket becomes a candidate.
  • There will be false positives and false negatives though.

Partition into Bands
• Trick: Divide the signature matrix 𝑀 into 𝑏 bands of 𝑟 rows.
   • For each band, hash each column to a hash table with 𝑘 buckets.
      • Make 𝑘 as large as possible to reduce collisions.
      • Use a different hash table for each band.
   • Becomes a candidate pair if hashed to the same bucket for ≥ 1 band.
   • Tune 𝑏 and 𝑟 to catch most similar pairs, but few non-similar pairs.

Example: False Positives
• Suppose that 𝐶" and 𝐶# are 40% similar (i.e., negative pair).
• False positives: Less than 1/5th of all 40%-similar pairs.
   • Probability identical in any one particular band: 0.4 % = 0.01.
   • Probability identical in ≥ 1 of 20 bands: 1 – 1 − 0.01 "& < 0.2.
• We can remove more than 80% of all 40%-similar pairs!

Example: False Negatives
• Suppose that 𝐶" and 𝐶# are 80% similar (i.e., positive pair).
• False negatives: About 1/3000th of all 80%-similar pairs.
   • Probability identical in any one particular band: 0.8 % = 0.328.
   • Probability not similar in all 20 bands: 1 − 0.328 "& = 0.00035.
• We can find almost all positive pairs even with LSH!

Distance Measures
• Generalized LSH is based on the distance between points.
   • Jaccard similarity is not a distance.
   • Jaccard distance = 1 – Jaccard similarity is a distance.
• A function 𝑑 is a distance measure if
   • It takes two points in the space and returns a real number satisfying:
      • 𝑑 𝑥, 𝑦 ≥ 0 (non-negative)
      • 𝑑(𝑥, 𝑦) = 0 iff 𝑥 = 𝑦 (distances are positive except for 𝑥 = 𝑦)
      • 𝑑 𝑥, 𝑦 = 𝑑(𝑦, 𝑥) (symmetric)
      • 𝑑 𝑥, 𝑦 ≤ 𝑑(𝑥, 𝑧) + 𝑑(𝑧, 𝑦) (triangle inequality)
