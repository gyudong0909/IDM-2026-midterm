Observations about Buckets
• Q: Why do we do that?
• A: To eliminate pairs that has no chance to be frequent.
  • Each bucket 𝐵 has a count of all pairs that hash to 𝐵.
     • If count ≥ support threshold 𝑠, pairs could all be frequent.
     • Otherwise, no pair can be frequent!
  • In Pass 2, define candidate pairs 𝐶! as 𝑖, 𝑗 such that:
     • Both 𝑖 and 𝑗 are frequent items.
     • 𝑖, 𝑗 hashes to a frequent bucket → Difference from A-Priori.
  • Later stages are similar to A-Priori.

Between Passes in PCY
• Replace the buckets by a bit-vector:
  • 1 means the bucket count exceeded the support threshold 𝑠.
  • 0 means it did not.
• The bitmap reduces the space by 1/32.
  • Since an integer is (normally) 32 bits.

One Subtlety
• PCY cannot use the triangular-matrix method for counting pairs.
  • Because PCY utilizes the sparsity of candidate pairs.
  • No way to “compact” a triangular matrix removing infrequent pairs.
• So PCY is always forced to use the triples method.

Extensions of PCY
• Two extensions of PCY: Multistage and Multihash algorithms.
   • Both create multiple hash tables to reduce # of candidate pairs.
• The way they create multiple tables is different:
   • Multistage: Create tables through successive passes.
   • Multihash: Create tables at the same time (i.e., in parallel).




                                     Jaemin Yoo                         13
The Multistage Algorithm
• Improves PCY by creating several successive hash tables:
  • Pass 1: Same as PCY; results in a bitmap.
  • Pass 2: Create the second hash table to generate another bitmap.
     • This table has 31/32 of buckets due to the first bitmap.
     • Here a pair 𝑖, 𝑗 is hashed only if
         • Both 𝑖 and 𝑗 are frequent.
         • The pair is hashed to a frequent bucket in Bitmap 1.
  • Idea: The second table can contain much fewer frequent buckets.

Limited-Pass Algorithms
• Many applications can sacrifice accuracy for speed.
  • E.g., it’s enough to find most frequent itemsets in a supermarket.
• We can find all or most frequent itemsets using ≤ 2 passes.
  • Random sampling: Simplest approach.
  • Toivonen: Two passes on average, but also may not terminate.
  • SON: Divide the data into multiple chunks.

Avoiding Errors in Sampling
• Sampling introduces both false positives and false negatives.
   • False positives: Itemsets frequent in sample, but not in the whole.
   • False negatives: Itemsets frequent in the whole, but not in sample.
• Eliminate false positives by making another full pass.
   • Count all frequent itemsets in sample.
• Reduce false negatives by reducing threshold (say to 0.9𝑝𝑠).
   • No easy way to eliminate false negatives.

Toivonen’s Algorithm
• Q: Can we eliminate false negatives at all?
• Idea: Construct negative border for an efficient safety check.
• Toivonen’s algorithm:
   • Use one pass over samples and one pass over the full data.
   • No false negatives or positives.
   • However, there is a small chance it will not finish by 2 passes.
      • The average number of passes to produce answer is still a constant.




                                         Jaemin Yoo                           27
Toivonen’s Algorithm (Pass 1)
• Find candidates from a small sample.
  • Use support threshold less than proportional value 𝑝𝑠 (say 0.9𝑝𝑠).
• Construct the negative border.
  • Collection of itemsets that are not frequent in sample, but all of their
    immediate subsets are frequent in sample.
  • Immediate subsets: Constructed by deleting exactly one item.

Toivonen’s Algorithm (Pass 2)
• Make a pass on the entire dataset.
   • Count itemsets (a) frequent in sample or (b) existing in negative border.
• Finish if no member of negative border is frequent in full data:
   • We have found the correct set of frequent itemsets
• Otherwise, there may be a larger set that is frequent.
   • Do not finish, and repeat the algorithm with new random sample.




                                     Jaemin Yoo                                  30
Why Toivonen’s Algorithm Works
• No false positives: Clearly because of Pass 2.
• No false negatives:
   • Suppose an itemset 𝑆 frequent in whole data, but not in sample.
   • Then, the border contains at least one itemset frequent in the whole.
   • Why? One of the following holds:
      • 𝑆 is in the negative order.
      • 𝑆 has an immediate subset 𝑇 frequent in whole data, but not in the sample.
          • This goes recursively until there is a subset in the negative order.
          • Recall that every singleton is in the negative order.

SON Algorithm
• Avoids both false negatives and positives with two full passes.
• Pass 1:
   • Divide input file into 1/𝑝 chunks.
   • Run A-Priori with 𝑝𝑠 as support threshold.
• Pass 2:
   • Take union of all frequent itemsets and select those with support ≥ 𝑠.
• There are no false negatives!
   • A frequent itemset must be frequent in at least one chunk.
   • If an itemset is not frequent in any chunk, its support is < (1/𝑝)𝑝𝑠 = 𝑠.


                                     Jaemin Yoo                                  33
SON with MapReduce
• MapReduce (or any other parallel computation) can be used.
• Pass 1: Find candidate itemsets.
   • Map: Take a chunk and return 𝐹, 1 for each frequent itemset 𝐹.
      • Support threshold is 𝑝𝑠.
   • Reduce: Ignore value and produce itemsets that appear at least once.
• Pass 2: Find true frequent itemsets.
   • Map:
      • Take candidate itemsets from Pass 1 and a portion of the input data.
      • Return 𝐶, 𝑣 where 𝐶 is a candidate itemset and 𝑣 is the support for this portion.
   • Reduce: For each itemset, sum the values and output if the sum ≥ 𝑠.
