📝 유형 1: MapReduce Pseudocode 작성 ★★★

출제 빈도: 매년 (closed-book 2026 거의 확실) · 관련 강의: L2 · 무엇을 묻나: 주어진 계산 작업을 Map / Reduce 함수로 직접 손코딩. Combiner 활용 여부도.

🔑 해결 패턴

기본 signature:
Map(key, value) → emit(k', v') 여러 개
Reduce(k', [v', ...]) → emit(k', result)

Combiner 조건: Reduce 함수가 commutative + associative일 때만 적용 가능

📚 정석 4가지 패턴 (암기 필수)

1. Word Count
Map(key, line):
    for word in line.split():
        emit(word, 1)
Reduce(word, counts):
    emit(word, sum(counts))
# Combiner OK (sum = comm + assoc)

2. Matrix × Vector (v가 메모리에 들어감)
Map((i,j), m_ij):
    emit(i, m_ij * v[j])     # v는 broadcast
Reduce(i, values):
    emit(i, sum(values))

3. Variance by Grade
Map(key, (sid, grade, score)):
    emit(grade, score)
Reduce(grade, scores):
    n    = len(scores)
    mean = sum(scores) / n
    var  = sum((s - mean)**2 for s in scores) / n
    emit(grade, var)
# Combiner 불가 — variance는 mean 의존

4. Mutual Friends (공통 친구)
Map(user, friends):
    for pair (u, v) in friends:
        emit(sorted((u, v)), user)
Reduce((u, v), mutuals):
    emit((u, v), mutuals)   # 공통 친구
📘 공식 20pt [2024 Midterm Q2(a)] Matrix-Vector with 6-number memory limit
📋 문제 원문 (as given on exam)
Compute Av using MapReduce, where v ∈ ℝ⁶ and A ∈ ℝ^(3×6). The memory can store only 6 numbers at a time. Describe the MapReduce process including use of a Combiner.
✅ 모범답안
v(6개)가 메모리에 들어감 → Case 1 (정석 패턴 2번). 
Map((i,j), m_ij):
    emit(i, m_ij * v[j])
Reduce(i, values):
    emit(i, sum(values))
Combiner 활용: sum = commutative + associative → Map에서 local partial sum 수행 → shuffle 데이터 대폭 감소.
예시: A=[[1..6],[7..12],[13..18]], v=[1..6] → Av = [91, 217, 343]
💭 풀이 해설
• 핵심: v를 모든 Mapper에 broadcast해서 메모리에 올려둠
• 각 Map은 m_ij와 대응 v[j]를 곱해 row i로 emit
• Reduce에서 같은 row 곱들을 sum
• Combiner 없으면 (i, val) 모두 shuffle → 네트워크 부하 큼
• Combiner 적용 시 row당 1개 값만 shuffle
📘 공식 5pt [2024 Midterm Q2(c)] Variance of Scores by Grade
📋 문제 원문
Using MapReduce, write pseudocode to calculate the variance of scores for each grade. Input: (Student ID, Grade, Score). Output: (Grade, Variance).
✅ 모범답안
Map(_, (sid, grade, score)):
    emit(grade, score)

Reduce(grade, scores):
    n    = len(scores)
    mean = sum(scores) / n
    var  = sum((s - mean)**2 for s in scores) / n
    emit(grade, var)
💭 풀이 해설
⚠️ "Combiner 쓸 수 있나?" 물으면 답: NO
• variance 계산은 mean 의존 → 모든 score를 모아야 함
• 일반 combiner는 (grade, score) 쌍을 합치는 게 의미 없음
• 단, 변형 combiner: (N, SUM, SUMSQ)를 분리 전달하면 가능 (BFR 스타일)
📘 공식 10pt [2024 Midterm Q2(b)] Why Map failure re-runs ALL, Reduce only in-progress?
📋 문제 원문
In MapReduce, when a Map worker fails, all tasks assigned to that worker need to be re-executed (including completed ones). When a Reduce worker fails, only in-progress tasks need to be re-executed. Explain why this difference exists.
✅ 모범답안
  • Map 출력은 worker 노드의 로컬 디스크에 저장 → 노드 실패 시 파일 접근 불가 → 완료된 task까지 전부 재실행.
  • Reduce 출력은 DFS (distributed file system, replicated)에 저장 → 노드 실패해도 출력 살아남음 → 진행 중 task만 재실행.
💭 풀이 해설
핵심: 두 출력의 저장 위치 차이.
• Map 중간결과 DFS에 저장하면 안정적이지만 I/O 폭증 → 로컬 디스크 선택 (성능 최적화)
• Reduce 최종결과는 durability 중요 → DFS에 저장 (안정성 최적화)
• 이 trade-off가 fault-tolerance 정책을 결정.

🛒 유형 3: Frequent Itemsets 시뮬레이션 ★★★

출제 빈도: 매년 (계산+증명 결합) · 관련 강의: L4, L5 · 무엇을 묻나: A-Priori / PCY / Toivonen 손으로 실행.

🔑 해결 패턴

A-Priori 손풀이 순서:

  1. Pass 1: 각 item support 카운트 → support ≥ s인 item = L₁
  2. Between: frequent item을 1..m으로 재번호
  3. Pass 2: frequent item끼리 pair만 카운트 → L₂
  4. 일반: L_{k-1} → C_k 구성 → support 카운트 → L_k 반복

PCY 추가: Pass 1 여유 메모리 → pair hash table → bitmap (1/32 공간) → Pass 2는 두 조건 (frequent items + frequent bucket)

Toivonen 추가: 샘플 + negative border → 전체 검증 → border가 frequent면 재시도

📘 공식 20pt [2024 Midterm Q3] Toivonen with Sample + Negative Border
📋 문제 원문
Given 8 transactions T1–T8 with items {A, B, C, D, E}, support threshold s = 3. Use Toivonen's algorithm with sample {T1, T2, T4, T5} and s' = 1. (i) Find frequent itemsets in the sample and construct the negative border. (ii) Prove there are no false negatives. (iii) Find T9, T10 that make the algorithm fail to terminate.
✅ 모범답안
(i) s'=1이면 샘플에 1번 이상 등장한 모든 itemset이 frequent.
Sample: T1={A,B,D}, T2={A,C,D,E}, T4={A,B,E}, T5={A,B,C}.
• Singletons: {A},{B},{C},{D},{E} 모두 frequent
• Pairs: {A,B},{A,C},{A,D},{A,E},{B,C},{B,D},{B,E},{C,D},{C,E},{D,E} 대부분 frequent
• Triples in sample: {A,B,D}(T1), {A,B,E}(T4), {A,B,C}(T5), {A,C,D}(T2), {A,C,E}(T2), {A,D,E}(T2), {C,D,E}(T2)
• Negative border: 샘플에 없지만 모든 immediate subset은 frequent. 예: {B,C,D} — 2-subset {B,C},{B,D},{C,D} 모두 frequent이나 실제 샘플에선 미출현.

(ii) No false negative: 전체에서 frequent인 itemset S가 있다면,
  • S의 모든 subset도 전체에서 frequent (monotonicity)
  • S가 샘플에서 frequent 아니라면, S 또는 S의 어떤 subset이 negative border에 있음
  • Pass 2에서 border까지 전체 데이터로 체크 → 놓친 S 발견

(iii) 실패 시나리오: negative border 원소가 전체에서 frequent이 되어야 함. 예: {B,C,D}가 border인데 T6={B,C,D} 이미 존재하고 T9={B,C,D}, T10={B,C,D} 추가하면 support ≥ 3 → 실패.
💭 풀이 해설
Toivonen의 핵심 아이디어: 샘플에서 놓친 frequent itemset은 반드시 negative border에 걸림. 이걸 전체 데이터로 검증하면 false negative 없음.
단점: border 원소가 실제 frequent이면 알고리즘 실패 → 재샘플링. 최악의 경우 수렴 보장 없음 (평균 상수 패스).

🔍 유형 4: MinHash 서명 + LSH Banding 계산 ★★★

출제 빈도: 2025 midterm 출제, 2026 범위 포함 · 관련 강의: L6, L7 · 무엇을 묻나: 서명 계산 + Jaccard 추정 + S-curve 튜닝.

🔑 MinHash 서명 알고리즘

sig[i][c] = ∞ for all hash i, column c
for each row r with value(c, r) = 1:
    for each hash i:
        sig[i][c] = min(sig[i][c], h_i(r))

근본 항등식 (증명 가능해야): P(h(C₁)=h(C₂)) = Jaccard(C₁, C₂).
증명: permutation 후 "첫 active row(둘 중 하나라도 1)"가 둘 다 1일 확률 = |∩|/|∪|.

🔑 LSH Banding 공식

P(candidate pair) = 1 - (1 - s^r)^b
Threshold t ≈ (1/b)^(1/r)
(k=r·b 고정) r↑: steep, precision↑ recall↓ | b↑: 완만, recall↑ precision↓

🔑 AND/OR Construction

AND (모두 일치): p_new = p^k
OR (하나라도): p_new = 1 - (1-p)^k
📘 공식 15pt [2025 Midterm Q2] Full LSH Pipeline — Finding Similar Items
📋 문제 원문 (요약)
3 documents, 6-character shingles, indexing function f → characteristic matrix. Apply 4 hash functions g1-g4 → compute minhash signatures. Compute true Jaccard and estimated Jaccard (from signatures) for each pair. Apply LSH banding with m=4 for various (r, b) configurations.
✅ 모범답안 (풀이 과정)
  1. Normalize 문서: lowercase, 구두점 제거
  2. 6-character shingle 추출, f(shingle) → row index
  3. Characteristic matrix: row=shingle idx, col=doc
  4. Hash 적용: g_i(r) → col별 min 유지 → signature
  5. True Jaccard = |∩|/|∪| (실제 shingle 집합)
  6. Estimated Jaccard = signature 일치 row 비율
  7. LSH banding (m=4): 예 r=2,b=2 → P(cand) = 1-(1-s²)²
💭 풀이 해설
전형적 파이프라인 문제. 각 단계를 순서대로 따라가며 표를 채우면 됨. 함정: signature 길이가 작으면(m=4) 추정 오차 큼 → true와 estimated Jaccard가 다르게 나올 수 있음.
📝 내 풀이 8pt [2021 Final Q4] LSH AND/OR Construction + Fixed Point ★
📋 문제 원문
Suppose we have an LSH family F of (d1, d2, 0.6, 0.4) hash functions. (a) Suppose we use two member functions from F and the AND-construction to form a (d1, d2, w, x) family. Calculate w and x. (b) Suppose we use two member functions from F and the OR-construction to form a (d1, d2, y, z) family. Calculate y and z. (c) Suppose we perform cascading where we do the OR-construction described in (b) and then the AND-construction described in (a). What is the fixedpoint p?
✅ 정답
(a) AND: w = 0.6² = 0.36, x = 0.4² = 0.16
(b) OR: y = 1 − (1−0.6)² = 0.84, z = 1 − (1−0.4)² = 0.64
(c) Cascade fixed point: p ≈ 0.38 (p = p²(2−p)² 만족)
💭 풀이 해설
(a) AND: 두 member 모두 충돌해야 → 확률 제곱
(b) OR: 하나라도 충돌 → 1 − (1 − p)²
(c) Cascade OR → AND:
• 입력 p → OR 후: p₁ = 1 − (1−p)² = p(2−p)
• AND 후: p₂ = p₁² = p²(2−p)²
• Fixed point: p = p²(2−p)². p≠0이면 1 = p(2−p)². 수치해 p ≈ 0.38.
의미: cascading으로 S-curve 더 가팔라짐 → 0.38 아래는 더 낮게, 위는 더 높게 → amplification.

🎯 유형 5: Clustering 시뮬레이션 ★★★

출제 빈도: 2025 midterm CURE+A-CURE 출제 · 관련 강의: L8, L9 · 무엇을 묻나: k-means++ 확률 / BFR Mahalanobis / CURE 시뮬. Novel Variant 대비 필수.

🔑 k-means++ D² 확률

Pr[pick p] = D(p)² / Σ_q D(q)²

D(p) = 이미 선택된 centroid와의 최소 거리. 제곱이 핵심 (outlier 가중치↑).

🔑 BFR Sufficient Statistics

N, SUM_d = Σ x_d, SUMSQ_d = Σ x_d²
centroid_d = SUM_d / N
σ_d² = SUMSQ_d / N − (SUM_d / N)²
mahal = √Σ_d ((x_d − c_d) / σ_d)² < 2√D → DS 배정

🔑 CURE 5단계

  1. sample 추출 → 메모리 내 hierarchical clustering
  2. 각 cluster에서 대표점 k개 선택 (centroid에서 가장 먼 점 → 그 다음 기존 rep에서 가장 먼 점)
  3. Shrink: p' = (1−α)p + αc (α ≈ 0.2)
  4. 나머지 점을 shrunk rep 기준 가장 가까운 cluster에 배정
📘 공식 20pt [2025 Midterm Q3] CURE Simulation + A-CURE Novel Variant ★★
📋 문제 원문 (요약)
(a) Given 2 clusters with points P1-P8. (i) Compute centroid of each cluster. (ii) Select 2 representative points per cluster. (iii) Shrink representatives 50% toward centroid. (iv) Assign new point P9 to a cluster. (b) A-CURE (Adjusted CURE): novel variant where shrinkage ratio α_i = 1/d_i. Compute moved representatives.
✅ 모범답안
(a) 표준 CURE:
  1. C1 centroid = (−2, 2), C2 centroid = (2, −3)
  2. 각 cluster에서 centroid에서 가장 먼 점 → 첫 rep. 첫 rep에서 가장 먼 점 → 두 번째 rep.
  3. Shrink 50%: p' = 0.5·p + 0.5·c (α = 0.5)
  4. P9와 모든 shrunk rep의 거리 계산 → 최소 거리의 cluster로 배정
(b) A-CURE: α_i = 1/d_i → 가까운 rep은 더 많이 shrink, 먼 rep은 덜 shrink.
p'_i = (1 − α_i)p_i + α_i·c = (1 − 1/d_i)p_i + (1/d_i)c
• 먼 rep (큰 d_i): α_i 작음 → 거의 이동 안 함, 원래 위치 유지
• 가까운 rep (작은 d_i): α_i 큼 → centroid 방향으로 많이 이동
💭 풀이 해설
Novel Variant 푸는 요령:
1. 원본 CURE의 shrink 공식을 새 α 정의로 대체
2. 새 공식 계산 (cluster별로)
3. 효과를 정성적으로 설명 (outlier 효과 변화 등)

2026 대비: 비슷한 "시험장 정의" 변형 대비. 각 알고리즘 파라미터 역할 이해 필수.

📐 유형 6: SVD 계산 (Projection, Energy) ★★

출제 빈도: 2021 final 출제, 2026 midterm 신규 범위(L10) · 관련 강의: L10 · 무엇을 묻나: SVD 명시적 계산 + projection + 유사도.

🔑 SVD 공식들

M_{m×n} = U_{m×r} Σ_{r×r} V^T_{r×n}
Energy total = Σ σ_i² = ||M||_F²
Retained by top-k = Σ_{i≤k} σ_i² / Σ σ_i²
||M − M_k||_F² = Σ_{i>k} σ_i²
new user q의 concept projection = q · V

📋 문제 원문

Its SVD: UΣV^T = (given) Suppose new user U5 has: 1 for M1, 2 for M3, 3 for M4. So U5 = [1, 0, 2, 3]. (a) What is the representation of U5 in the concept space? (b) Which user among U1-U4 is most similar to U5 in cosine distance in concept space? (c) Calculate cosine of the angle between U5 and the closest user in concept space.
✅ 정답
(a) U5 in concept space = [3, √2/2, 2]
(b) 가장 유사: U4
(c) cos(U5, U4) = 12 / (4√13.5) = 3/√13.5 ≈ 0.816
💭 풀이 해설
(a) concept space projection = q · V.
V의 각 column (V^T의 row를 전치):
• V col1 = [0, 0, 0, 1]^T
• V col2 = [√2/2, √2/2, 0, 0]^T
• V col3 = [0, 0, 1, 0]^T
U5 · V:
• entry 1: 1·0 + 0·0 + 2·0 + 3·1 = 3
• entry 2: 1·(√2/2) + 0·(√2/2) + 2·0 + 3·0 = √2/2

👥 유형 7: Collaborative Filtering 계산 ★★

출제 빈도: 2022, 2024 final 출제 · 관련 강의: L12 · 무엇을 묻나: 시간복잡도 / Pearson / 추천 설계.

🔑 Pearson Correlation

sim(x,y) = Σ(r_xi − r̄_x)(r_yi − r̄_y) / √[Σ(r_xi − r̄_x)²] √[Σ(r_yi − r̄_y)²]
(공통 평가 아이템 S_xy에 대해서만)

🔑 예측 (k-NN weighted)

r̂_xi = r̄_x + Σ sim(x,y)·(r_yi − r̄_y) / Σ |sim(x,y)|
📝 내 풀이 6pt [2022 Final Q2(a,b)] CF 시간복잡도
📋 문제 원문
Given a utility matrix with N users and M items: (a) In terms of N and M, what is the time complexity of predicting the ratings of a customer using user-user collaborative filtering? (b) What is the time complexity of predicting the ratings of a customer using item-item collaborative filtering?
✅ 정답 (추정)
(a) User-user: O(NM)
(b) Item-item: online O(kM) (pre-compute 전제), offline pre-compute는 O(M²N)
💭 풀이 해설
(a) User-user:
  • target user x에 대해 모든 N−1 유저와 similarity 계산. 각 similarity는 M 아이템 비교 → O(NM).
  • 그 후 M 아이템 예측, 각 예측에 top-k 유저 참조 → O(kM).
  • 총 O(NM + kM) ≈ O(NM).
(b) Item-item:
  • 예측할 아이템 i마다, x가 평가한 아이템(~M)과 similarity 계산. 각 sim은 N 유저 비교 → O(M·N) per prediction.
  • M 아이템 예측 시 O(M²N) — 하지만 item similarity는 오프라인 pre-compute 가능.
  • 온라인만: O(kM) (pre-compute된 similarity로 top-k 참조).
📝 내 풀이 12pt [2024 Final Q2] Diversity-aware Recommender (open-ended)
📋 문제 원문
Diversity is an important aspect to evaluate a recommender system. That is, we want each user to get a list of recommendation consisting of diverse items (e.g., we don't want the list to be filled with all horror movies). Suppose that we are given only the utility matrix R ∈ [1,5]^(N×M), where N is the number of users and M is the number of items. Describe your own way to create a recommender system that supports high diversity, with your own definition of diversity.
✅ 내 풀이 (예시 답안)
Diversity 정의: 추천 리스트 S에 대해
diversity(S) = (1 / |S|(|S|-1)) · Σ_{i,j ∈ S, i≠j} (1 − sim(i, j))
(리스트 내 모든 pair의 평균 item-item cosine distance)

추천 알고리즘 — Greedy MMR (Maximal Marginal Relevance):
  1. user 평점 → SVD/Pearson으로 item-item similarity 행렬 S 오프라인 구축
  2. user u의 각 item i에 relevance(u, i) 예측 (item-item CF)
  3. 추천 리스트 L은 greedy하게 선택:
    score(i) = λ·relevance(u, i) − (1−λ)·max_{j ∈ L} sim(i, j)
  4. 최초 가장 relevant item 선택, 이후 score(i) 최대화 — 기존 선택과 덜 유사한 걸 선호
  5. λ = 0.7 (trade-off 파라미터)
💭 풀이 해설
Open-ended이므로 설득력 있는 자신만의 정의 + 구체 알고리즘 필요.
핵심 포인트: (1) diversity를 수식으로 정의 (2) relevance vs diversity trade-off (λ) (3) item-item similarity 재사용
다른 접근: Determinantal Point Process (DPP), clustering 기반 분산 등도 OK.

🧮 유형 8: UV Decomposition + SGD Update ★

출제 빈도: T/F 위주, 직접 계산은 드묾 · 관련 강의: L13 · 무엇을 묻나: SGD 1-step 업데이트 + BPR.

🔑 암기 필수 공식

r̂ = u_x^T v_i  (bias 포함: r̂ = μ + b_x + b_i + u^T v)
SSE = Σ_{(x,i) ∈ obs} (r_xi − u_x^T v_i)² + λ(||U||² + ||V||²)

SGD update 3줄:
ε = 2(r − u^T v)
u ← u + η(ε·v − λ·u)
v ← v + η(ε·u − λ·v)

BPR pairwise loss: Σ_{(x,i,j)} −log σ(u_x^T v_i − u_x^T v_j)
📝 내 연습문제 5pt 연습: SGD 1-step (practice problem)
📋 연습 문제
u = [1, 0.5], v = [2, -1], observed rating r = 3, learning rate η = 0.1, regularization λ = 0.01. Perform one SGD step. What are u_new and v_new?
✅ 정답
u_new = [1.599, 0.1995]
v_new = [2.298, -0.849]
💭 풀이 해설
1. 예측값 계산:
û = u^T v = 1·2 + 0.5·(−1) = 1.5

2. 오차 ε:
ε = 2(r − û) = 2(3 − 1.5) = 3

3. u 업데이트:
u_new = u + η(ε·v − λ·u)
= [1, 0.5] + 0.1·(3·[2,−1] − 0.01·[1, 0.5])
= [1, 0.5] + 0.1·([6, −3] − [0.01, 0.005])
= [1, 0.5] + [0.599, −0.3005]
= [1.599, 0.1995]

4. v 업데이트:
v_new = v + η(ε·u − λ·v)
= [2, −1] + 0.1·(3·[1, 0.5] − 0.01·[2, −1])
= [2, −1] + 0.1·[2.98, 1.51]
= [2.298, −0.849]

📖 MMDS Ex 1.2.1 — Bonferroni 원칙 변형
📋 문제
원본 문제: N명의 사람, D일간 호텔 방문 패턴. 같은 호텔에 같은 날 있었던 pair 추정 (원본 ≈ 250,000). (a) 일수 D가 1000 → 2000으로 증가하면? (b) 인구 10억 → 20억, 호텔 10만 → 20만이면? (c) 기준이 "최소 3일" 같은 호텔로 강화되면?
✅ 풀이
(a) D² / D = D 비례 → 2배 ≈ 500,000. day-pair 수는 C(D,2) ∝ D²로 증가, 분모도 D로 증가.
(b) N² 4배 × (호텔 2배로 확률 1/2) = 2배 ≈ 500,000.
(c) 3일 매칭 확률 p³ = 10⁻⁹. 기대 pair수 ≈ 0.08 ≈ 0. Bonferroni 원칙: 플래그 안 해야.
📖 MMDS Ex 1.2.2 — Market-basket Bonferroni
📋 문제
1000 items, 10⁹ baskets (각 크기 10). 1% 이상 basket에 공동 등장하는 item pair를 찾음. 이 기준이 의미 있는가?
✅ 풀이
• 랜덤 basket에 특정 item 포함 확률 = 10/1000 = 1/100
• 특정 pair {i,j} 공동 등장 확률 = (1/100)² = 10⁻⁴
• 기대 basket 수 = 10⁹ × 10⁻⁴ = 10⁵
• threshold = 10⁹ × 1% = 10⁷
• 10⁵ ≪ 10⁷ → 랜덤으론 threshold 못 넘음 → frequent pair는 의미 있음 ✓
📖 MMDS Ex 1.3.1 — IDF 계산
📋 문제
검색 엔진에 10⁷개 문서 인덱싱됨. IDF(w) = log₂(N/n_w). (a) 단어 w가 40개 문서에 나옴 → IDF? (b) 단어 w가 10,000개 문서에 나옴 → IDF?
✅ 풀이
(a) IDF = log₂(10⁷/40) = log₂(2.5×10⁵) ≈ 18
(b) IDF = log₂(10⁷/10⁴) = log₂(1000) ≈ 10
→ 자주 등장하는 단어일수록 IDF 낮음 (정보량 적음)
📖 MMDS Ex 1.3.3 — Hash 함수가 permutation이 되는 조건
📋 문제
h(x) = cx mod 15 가 {0,...,14}의 bijection이 되는 c 값들을 찾아라.
✅ 풀이
gcd(c, 15) = 1 이어야 함. 15 = 3×5이므로 c가 3이나 5의 배수면 안 됨.

📘 Ch 2: MapReduce (L2, L3)

📖 MMDS Ex 2.2.1 — Word-count에서 Load Balance
📋 문제
(a) 10 reducers — 어떻게 skew가 발생? (b) 10,000 reducers — skew 줄어드나? (c) Combiner를 쓰면 어떻게 바뀌나?
✅ 풀이
(a) 단어 빈도는 Zipf 법칙 (power law). "the", "a" 같은 고빈도 단어 담당 reducer는 훨씬 많은 KV pair 처리 → 심각한 skew.
(b) 각 reducer가 담당하는 vocabulary 비율은 작아지지만, 고빈도 단어의 reducer는 여전히 불균형. 절대적으론 나아지나 완전 해결 X.
(c) Combiner는 Mapper output에서 단어별 local count 집계. 각 reducer가 단어당 받는 값 수 = Mapper 수로 bounded. 고빈도 단어도 최대 1값/mapper → skew 대폭 감소.
📖 MMDS Ex 2.3.1 — 기본 연산의 MapReduce 구현
📋 문제
MapReduce로 구현: (a) 최댓값 (b) 평균 (c) 중복 제거 (d) distinct 개수
✅ 풀이
(a) 최댓값
Map(k, v): emit(k, v)
Reduce(k, values): emit(k, max(values))

(b) 평균
Map(_, x): emit(1, (x, 1))        # 단일 키로
Reduce(1, pairs):
    total, count = sum of first, sum of second
    emit(1, total/count)
# Combiner 가능: (partial_sum, partial_count) 쌍 전달

(c) 중복 제거
Map(_, x): emit(x, null)
Reduce(x, _): emit(x, null)       # 같은 키가 여러 번 와도 1번만

(d) distinct 개수
# 2-pass 필요
Pass 1: (c)와 동일 → distinct values
Pass 2: Map(x, _): emit(1, 1) / Reduce(1, values): emit(1, sum)
📖 MMDS Ex 2.3.2 — Matrix × Vector (MapReduce 핵심)
📋 문제
M = r×c 행렬, v = 길이 c 벡터. Mv를 MapReduce로 계산.
✅ 풀이
Map: 각 m_ij에 대해 emit(i, m_ij · v_j)
Reduce: 행 i마다 값들 합산 → emit(i, sum)
결과: (i, (Mv)_i = Σ_j m_ij v_j). v는 모든 Mapper에 broadcast 필요 (메모리에 들어가는 경우).
📖 MMDS Ex 2.3.3 — Bag에서 관계대수 (Union, Intersection, Difference)
📋 문제
Bag(multiset) R, S에 대해 R∪S, R∩S, R-S를 MapReduce로.
✅ 풀이
Map 공통: R의 tuple t → (t, 'R'), S의 tuple t → (t, 'S').
같은 tuple이 R에 m_R번, S에 m_S번 등장 가정. 
Union:       Reduce → output t 총 (m_R + m_S) 번
Intersection: Reduce → output t 총 min(m_R, m_S) 번
Difference:   Reduce → output t 총 max(0, m_R - m_S) 번

📖 MMDS Ex 3.5.1 — 거리 함수 체크 ★
📋 문제
다음 중 어느 것이 distance measure인가? (metric 공리 4개 확인) (a) d(x,y) = max(x, y) (비음수 실수) (b) d(x,y) = |x-y| (실수) (c) d(x,y) = x + y (비음수 실수)
✅ 풀이
(a) max(x,y): NOT distance
→ d(3,3) = 3 ≠ 0. 공리 (2) 위배 (자기 자신과의 거리는 0).

(b) |x-y|: Valid distance ✓

📖 MMDS Ex 3.6.1 — AND/OR Construction 조합
📋 문제

✅ 풀이
Case 1: AND(r) → OR(b) — 표준 LSH banding
• AND 후: p^r
• OR 후: 1 - (1 - p^r)^b
→ 표준 S-curve p(s) = 1 - (1 - s^r)^b

Case 2: OR(b) → AND(r)
• OR 후: 1 - (1 - p)^b
• AND 후: [1 - (1 - p)^b]^r
→ 다른 S-curve

비교: Case 1 (AND→OR)이 보통 더 가파른 threshold. LSH banding 표준 선택.

r̂_ui = r_ui - r̄_u
sim(u1, u2) = r̂_u1 · r̂_u2 / (||r̂_u1|| · ||r̂_u2||)

u_ik ← u_ik + 2α · e_ij · v_jk
v_jk ← v_jk + 2α · e_ij · u_ik

1. 현재 예측 (UV^T)_ij 계산

Theory · 이론

핵심 개념 (Key Concepts)

TF-IDF (Term Frequency - Inverse Document Frequency)

\[\text{TF-IDF}(i, j) = \text{TF}(i, j) \times \text{IDF}(i)\] \[\text{TF}(i, j) \propto \text{# occurrences of term } i \text{ in document } j\] \[\text{IDF}(i) \propto \frac{1}{\text{# documents where term } i \text{ appears}}\]

직관: TF-IDF가 높음 = 단어 i가 이 문서에는 자주 나오지만 전체 문서에선 드묾. "the" 같은 단어는 TF↑ IDF↓. 전문 jargon은 TF↓ IDF↑. 주제어는 둘 다 ↑.

Bonferroni's Principle (본페로니 원칙)

데이터에서 패턴을 찾을 때, 같은 크기의 랜덤 데이터와 비교하라. 랜덤에서도 예상되는 횟수가 크다면 그 패턴은 의미 없음 (bogus).

생일 문제: 23명이 모여도 생일이 겹칠 확률 > 0.5. 신기해 보이지만 통계적으로 자연스러움.

Hash Functions (해시 함수)

해시 함수 \(h\)는 key를 bucket \(\in [0, B-1]\)로 매핑. 대용량 데이터에서 O(1) 자료구조 가능. \(B < \text{unique keys 수}\)이면 충돌 불가피.

Power Laws (멱법칙)

\[\log y = a \log x + b \quad \Leftrightarrow \quad y = c \cdot x^a\]

log-log 스케일에서 선형. 예: 상품 판매량, 웹사이트 크기, 부의 분포 (80/20 법칙).

Memory Hierarchy (메모리 계층)

L0 Register > L1 Cache (SRAM) > L2/L3 Cache > Main Memory (DRAM) > Local Storage (SSD/HDD) > Remote/Cloud Storage (DFS). 위로 갈수록 빠르지만 작고 비쌈. 데이터가 어느 계층에 있느냐가 알고리즘 병목을 결정.

Example · 예시

  • ML 유용: 무엇(WHAT)을 원하는지는 알지만 어떻게(HOW) 할지 모를 때 — 추천, 예측, 임베딩
  • ML 불필요: 목표를 직접 기술할 수 있을 때 — 웹에서 이력서 위치 찾기, HTML 파싱

L2: MapReduce (분산 컴퓨팅)

Intuition · 직관

왜 분산 컴퓨팅이 필요한가?

"We need to bring computation to data."
연산을 데이터가 있는 곳으로 가져갑니다. chunk server에 알고리즘을 분산시켜서 데이터를 주고받지 않아요. — Prof. Yoo
"MapReduce is basically a style of programming. It is not a solution to every problem."
MapReduce는 프로그래밍 스타일이지, 모든 문제의 해결책이 아닙니다. — Prof. Yoo

핵심 통찰: 서버 1000대 → 하루 1개 실패 예상. 100만 대면 하루 1000개 실패. 체계적 fault tolerance 필수.

Theory · 이론

Distributed File System (DFS, 분산 파일 시스템)

  • Chunk nodes (data nodes): chunk 단위 (16-64 MB)로 저장, 2-3배 복제 (다른 rack의 노드에)
  • Master node (name node): 메타데이터 관리, 백업 있음
  • 사용 패턴: 거대한 파일, 읽기·append 주로, 제자리 수정은 드묾

MapReduce 구성 요소

\[\text{Map}: (k, v) \to \{(k_i, v_i)\}_i\] \[\text{Group by Key}: \{(k, v)\} \to (k, [v_1, v_2, \ldots, v_n])\] \[\text{Reduce}: (k, [v_1, \ldots, v_n]) \to (k, v)\]

3단계:

  1. Map: 사용자가 작성한 함수를 각 입력 원소에 적용. 출력: key-value 쌍들. 완전 병렬 가능.
  2. Group by Key: 시스템이 같은 key끼리 모음 (개념적 단계, hash 기반 shuffling으로 수행). 프로그래머가 쓰지 않음.
  3. Reduce: 사용자 함수를 (key, value list) 쌍에 적용. 값을 집계.

Combiner 최적화

Reduce 함수가 commutative (\(a+b=b+a\)) AND associative (\((a+b)+c=a+(b+c)\))일 때만. 값들을 어느 순서로든 합칠 수 있음 → Map 단계에서 local 집계 가능 → shuffle 대폭 감소. 프레임워크는 이 조건을 자동으로 확인하지 않음 (프로그래머 책임).

노드 실패 처리

실패 유형조치이유
Master 실패전체 job 재시작master 상태 복제 너무 비쌈
Map worker 실패완료된 task 포함 전부 재실행Map 출력은 로컬 디스크 → 노드 실패 시 접근 불가
Reduce worker 실패진행 중 task만 재실행완료된 reduce 출력은 DFS (복제됨)에 저장

Matrix-Vector 곱 in MapReduce

Case 1: v가 메모리에 들어감

// Map: input key=(i,j), value=m_ij
map((i,j), m_ij):
    emit(i, m_ij * v[j])     // v는 메모리에 있음 (broadcast)

// Reduce: 같은 row i의 모든 값을 합
reduce(i, values):
    emit(i, sum(values))

Case 2: v가 메모리에 안 들어감 — M을 세로 stripe로, v를 가로 stripe로 나눔. 각 Map task가 M의 한 stripe와 그에 맞는 v의 stripe를 처리.

Reduce task 개수

경험칙: # 노드 < # Reduce task < # key. 이론상 key당 1개 Reduce가 최대 병렬이지만 비현실적.

Example · 예시

Word Count — 전체 흐름

// Map function
map(key, value):                // key: line index, value: line text
    for each word w in value:
        emit(w, 1)

// Reduce function
reduce(key, values):           // key: word, values: [1, 1, ..., 1]
    result = 0
    for each count v in values:
        result += v
    emit(key, result)

Combiner 적용: 각 Map task가 local에서 (w,1),(w,1),(w,1) → (w,3)으로 합친 뒤 shuffle. shuffle 양 급감.

Mutual Friends (수업 퀴즈)

// Input: A -> B, C, D (A is friends with B, C, D)
map(key, friends):             // key: person, friends: friend list
    for each pair (w, v) in friends:
        emit((w,v), key)       // pair of friends -> original person

// Reduce: identity (just collect)
reduce((w,v), values):
    emit((w,v), values)        // output mutual friends list

Wide vs Narrow Dependency

  • Narrow (좁음): 각 파티션이 최대 1개 부모 파티션에 의존 → 병렬 빠름
  • Wide (넓음): 각 파티션이 여러 부모 파티션에 의존 → shuffle 필요, 느림
  • flatMap은 Narrow: 한 입력에서 여러 출력이 나와도, 각 출력 파티션은 단일 입력 파티션에만 의존

Blocking Property (DAG의 fault tolerance)

연산은 모든 선행이 끝난 뒤에만 실행. task 실패 시 후속이 시작 안 됐으므로 master가 정확히 뭘 재시작할지 앎.

MapReduce 대비 두 가지 개선

  1. Lazy Evaluation (지연 평가): action 호출 전엔 transformation 적용 안 함. 실행 계획 최적화, transformation 합성, 인메모리 전달 가능.
  2. Lineage (계통): 각 RDD가 부모 RDD를 기록. chunk 유실 시 lineage를 따라 올라가 재계산. chunk 단위로 추적.

L4: Frequent Itemsets 1 (빈발 아이템셋, A-Priori)

Intuition · 직관

왜 Frequent Itemsets를 찾는가?

"In a supermarket, the manager wants to find patterns of items. If someone buys diaper and milk, he is likely to buy beer."
슈퍼마켓에서 매니저가 아이템 패턴을 찾고 싶어합니다. 기저귀+우유를 사면 맥주도 살 가능성이 높아요. 기저귀 옆에 맥주 6-pack을 둬도 놀라지 마세요! — Prof. Yoo
"Market-basket model is an abstract of various data. Items do not have to be 'in' baskets!"
Market-basket 모델은 다양한 데이터의 추상화입니다. 아이템이 '바구니 안에' 있을 필요는 없어요. — Prof. Yoo

핵심 문제: Frequent itemset을 찾는 게 association rule 찾는 것보다 더 어려움. Itemset만 찾으면 rule 생성은 쉬움.

왜 pair가 가장 어려운가: itemset 크기가 커질수록 frequent일 확률이 exponentially 감소. Frequent pair는 많고, triple은 드묾.

Theory · 이론

Market-Basket Model (장바구니 모델)

  • Items: 모든 대상의 집합 (예: 가게의 상품들)
  • Baskets: items의 subset (예: 한 고객의 구매 내역)
  • 추상화: 모든 many-to-many 매핑에 적용. 주제 발견 (items=단어, baskets=문서), 표절 탐지 (items=문서, baskets=문장), 바이오마커 (items=약물, baskets=환자)

Key Definitions

\[\text{Support}(I) = \text{# baskets containing all items in } I\] \[\text{Itemset } I \text{ is frequent if } \text{Support}(I) \geq s\]

Association Rules: \(I \to J\) where \(I \cap J = \emptyset\)

\[\text{Support}(I \to J) = \text{Support}(I \cup J)\] \[\text{Confidence}(I \to J) = \frac{\text{Support}(I \cup J)}{\text{Support}(I)} = P(J|I)\] \[\text{Interest}(I \to J) = \text{Confidence}(I \to J) - P(J) = P(J|I) - P(J)\]

Interest 해석: 0에 가까움 = 규칙이 흥미롭지 않음 (I와 상관없이 J가 자주 나타남). 양수 = 양의 상관. 음수 = 음의 상관. 목표: support ≥ s AND confidence ≥ c인 rule.

Association Rule 탐색 (2단계)

  1. Step 1: 모든 frequent itemset I 찾기 (어려운 부분)
  2. Step 2: 각 subset \(A \subset I\)에 대해 \(A \to I \setminus A\) 규칙 생성. Confidence 체크. LHS에서 RHS로 아이템을 옮기면 confidence는 감소만 함 (분자 고정, 분모는 감소만). 이걸로 prune.

Closed vs Maximal Frequent Itemsets

개념정의장점단점
Closed즉시 superset 중 같은 support를 가진 게 없음안전한 압축, support 정보 유지여전히 많음
Maximal즉시 superset 중 frequent가 없음더 공격적 pruningsubset의 support 정보 손실

모든 maximal은 closed지만, 역은 성립하지 않음.

Pair를 메모리에서 카운팅

방식메모리유리한 경우
Triangular Matrix\(4 \times \frac{n(n-1)}{2}\) bytesDense data (p > 1/3)
Triples (i, j, count)\(12 \times p \times \frac{n(n-1)}{2}\) bytesSparse data (p < 1/3)

A-Priori 알고리즘

핵심 아이디어: Monotonicity (단조성) — itemset I가 ≥ s번 나타나면 모든 subset J도 ≥ s번 나타남. 대우: item i가 infrequent면 i를 포함하는 어떤 pair도 frequent일 수 없음.

2-Pass 알고리즘:

  1. Pass 1: basket을 읽고 각 item의 발생 횟수 카운트. 메모리 O(n). ≥ s인 item을 frequent로 표시.
  2. Between passes: frequent item을 1..m으로 재번호. non-frequent는 0.
  3. Pass 2: basket 다시 읽음. 각 basket에서 frequent item만 봄. frequent item 사이의 모든 pair 카운트. 메모리 O(m²).

일반화 파이프라인:

\[C_1 \xrightarrow{\text{Filter}} L_1 \xrightarrow{\text{Construct}} C_2 \xrightarrow{\text{Filter}} L_2 \xrightarrow{\text{Construct}} C_3 \xrightarrow{\text{Filter}} L_3 \to \cdots\]

\(C_k\) = candidate k-tuple, \(L_k\) = 실제로 frequent인 k-tuple. 각 Filter는 디스크 패스 필요. Construct는 필요 없음.

Example · 예시

A-Priori 완전한 예시

Items = {milk, coke, pepsi, beer, juice}, s = 3, 8 baskets:

B1={m,c,b}, B2={m,p,j}, B3={m,b}, B4={c,j}, B5={m,p,b}, B6={m,c,b,j}, B7={c,b,j}, B8={b,c}

Pass 1: item 카운트: m=5, c=5, p=2, b=6, j=4. Frequent (≥3): m, c, b, j. Pepsi 제거.

Pass 2: {m,c,b,j}의 pair만 카운트:

Pair{m,c}{m,b}{m,j}{c,b}{c,j}{b,j}
Count342432

Frequent pairs (≥3): {m,c}, {m,b}, {c,b}, {c,j}

Support / Confidence / Interest 예시

Rule: {m,b} → c. Support({m,b,c}) = 2. Support({m,b}) = 4.

Confidence = 2/4 = 0.5. P(c) = 5/8. Interest = 0.5 − 5/8 = −1/8 (음수!)

Exam Points · 시험 포인트

L4 필수 암기

  • 공식: Support, Confidence, Interest 정의
  • A-Priori monotonicity: I가 frequent이면 모든 subset도 frequent
  • 대우: item i가 infrequent면 i를 포함하는 어떤 superset도 frequent 아님
  • Triangular vs Triples: p < 1/3이면 Triples 유리
  • Closed vs Maximal: 모든 maximal은 closed. Maximal은 support 정보 손실.
  • Frequent itemset만 찾으면 rule은 쉽게 생성
  • 비용은 disk pass 수로 측정, 알고리즘 복잡도 아님

- "A-Priori finds interesting rules" → FALSE (frequent itemset 찾음, interest 아닌 support 사용)

L5: Frequent Itemsets 2 (PCY, Multistage, Toivonen, SON)

Intuition · 직관

왜 A-Priori를 개선하는가?

"The A-Priori algorithm is fine as long as counting candidate pairs can be done in main memory. What if the memory is not enough?"
A-Priori는 candidate pair 카운팅이 메인 메모리에 들어가면 괜찮은데, 메모리가 부족하면요? — Prof. Yoo
"In many data mining algorithms, we care about the trade-off between accuracy or approximation and speed."
많은 데이터 마이닝 알고리즘에서 정확도/근사 vs 속도 trade-off가 중요합니다. — Prof. Yoo

문제: A-Priori Pass 1 필터링 후에도 candidate pair C₂가 메모리 초과 가능. |C₂|를 더 줄이는 알고리즘 필요.

Theory · 이론

1. PCY (Park-Chen-Yu) Algorithm

동기: A-Priori Pass 1에서 대부분 메모리가 놀고 있음 (item count만 저장). 이 유휴 메모리를 pair hash table에 활용.

동작 방식:

  1. Pass 1: item 카운트 (A-Priori 동일) AND 모든 pair {i,j}를 bucket에 해시, bucket count 증가.
  2. Between passes: hash table을 bit-vector (bitmap)로 변환. 1 = bucket count ≥ s ("frequent bucket"), 0 = 아님. 1/32로 공간 절약 (integer → 1 bit).
  3. Pass 2: Candidate pair {i,j}는 두 조건 모두 만족해야:
    • (1) i, j 둘 다 frequent item
    • (2) {i,j}가 bitmap의 frequent bucket으로 해시

PCY는 triples method 필수 (triangular matrix 아님) — candidate pair의 sparsity를 활용하기 위해.

2. Multistage Algorithm

아이디어: 여러 패스에 걸쳐 서로 다른 hash 함수로 연쇄적인 hash table 생성.

  1. Pass 1: PCY와 동일 → Bitmap 1 생성
  2. Pass 2: Bitmap 1 + frequent item 사용. 조건 (1)+(2) 만족하는 pair만 새 hash table에 해시. 두 번째 table은 Bitmap 1이 공간 차지하므로 31/32 bucket. → Bitmap 2 생성
  3. Pass 3: 세 조건 모두 만족하는 pair만 카운트:
    • (1) 두 item 모두 frequent
    • (2) Bitmap 1의 frequent bucket에 해시
    • (3) Bitmap 2의 frequent bucket에 해시
결정적 함정 (시험 출제 단골): 조건 2 (Bitmap 1)를 버릴 수 없음 — Bitmap 2가 있어도 불가. Pass 2에서 조건 2 때문에 카운트 안 된 pair가 Bitmap 2에선 우연히 frequent bucket에 갈 수도 있음. 조건들이 독립적.

Truly frequent pair는 항상 frequent bucket에 해시 (몇 패스든).

3. Multihash Algorithm

아이디어: 단일 패스에서 여러 hash table 동시 생성 (순차가 아닌 병렬).

  • 각 table은 다른 hash 함수 사용, 같은 패스 공유
  • Pair {i,j} candidate 조건: 두 item 모두 frequent AND 모든 hash table에서 frequent bucket에 해시
  • Trade-off: table 많을수록 조건 많지만 각 table 작아 충돌↑
  • 최적 table 수: \(p^n\) 최소화 (p = infrequent pair가 frequent bucket에 해시될 확률)
\[P(\text{infrequent pair in } C_2) = p^n\] \[\text{Increasing } n \text{ increases both } n \text{ and } p. \text{ Optimal } n \text{ exists.}\]

4. Limited-Pass 알고리즘 (샘플링 기법)

Random Sampling

  • basket을 p 비율로 샘플링, threshold ps로 메모리 내 A-Priori 실행
  • False positive: full pass 한 번 (전체에서 검증)으로 제거 가능
  • False negative: 제거 어려움. threshold를 0.9ps로 낮춰 줄일 수 있음

Toivonen's Algorithm

  • Pass 1: 샘플에서 frequent itemset 찾기 (threshold < ps, 예: 0.9ps). Negative border 구성.
  • Negative border: 샘플에서 frequent 아니지만 모든 immediate subset은 frequent인 itemset 집합.
  • Pass 2: (샘플-frequent + negative border)를 전체 데이터에서 검증.
  • border item 중 전체에서 frequent인 게 없으면 → 완료 (no FP, no FN).
  • border item 중 하나라도 전체에서 frequent면 → 실패, 재샘플링.
  • No false negative 증명: S가 전체에서 frequent인데 샘플에선 아니라면, S가 negative border에 있거나, S의 immediate subset T가 "전체에선 frequent, 샘플에선 아님"이어야 함. 재귀적으로 border에 도달.
  • 평균 패스 수는 상수지만 deterministic 아님.

SON (Savasere, Omiecinski, Navathe) Algorithm

  • Pass 1: 데이터를 1/p 청크로 나눔. 각 청크에서 threshold ps로 A-Priori 실행.
  • Pass 2: 모든 청크의 frequent itemset 합집합. threshold s로 전체에서 검증.
  • No false negative (pigeonhole 증명): global support ≥ s라면 최소 한 청크에서 support ≥ ps. (모든 청크에서 < ps라면 총 < (1/p)·ps = s 모순.)
  • No false positive: Pass 2에서 검증.

SON with MapReduce

// Pass 1: Find candidate itemsets
map(chunk):                              // each mapper gets one chunk
    run A-Priori on chunk with threshold ps
    for each frequent itemset F:
        emit(F, 1)

reduce(F, counts):                       // collect all candidates
    if F appears at least once:
        output F as candidate

// Pass 2: Find true frequent itemsets
map(candidates, data_portion):           // each mapper gets candidates + data chunk
    for each candidate C:
        count support of C in this portion
        emit(C, local_support)

reduce(C, supports):                     // sum supports across all portions
    if sum(supports) >= s:
        output C as truly frequent

예상 빈칸: Pass 1 Map의 "emit(F, 1)", Pass 2 Reduce의 "sum(supports) >= s"

L6: Finding Similar Items 1 — Shingling & MinHash

Intuition · 직관

왜 LSH인가?

"Comparing every pair is O(N²). For 1B documents, impossible. Trade a small amount of accuracy for huge speedup."
모든 pair 비교는 O(N²). 10억 문서에선 불가능. 정확도 조금 희생하고 대폭 속도 향상.

목표: 많은 문서 중 Jaccard 유사도가 threshold s를 넘는 pair를 찾되 — 모든 pair 비교 없이.

3단계 파이프라인: Shingling → Min-Hashing → LSH (banding).

Theory · 이론

Step 1: Shingling

  • k-shingle: 문서 안의 길이 k인 부분문자열 (character 또는 word 단위)
  • 문서 → shingle 집합 → 집합 기반 유사도 가능
  • 경험칙: 짧은 문서(이메일)는 k=5, 큰 코퍼스는 k=9
  • 선택: shingle을 integer로 해시 (공간 절약, 각 4 bytes)

Jaccard Similarity

\[\text{sim}(C_1, C_2) = \frac{|C_1 \cap C_2|}{|C_1 \cup C_2|}\]

여기서 \(C_1, C_2\)는 shingle 집합. 범위 \([0, 1]\).

Characteristic Matrix (특성 행렬)

Row = shingle (universe), Column = 문서. \(M_{ij} = 1\) iff shingle \(i\)가 문서 \(j\)에 있음. 일반적으로 매우 sparse. 실제로 저장하지 않음 — minhash signature로 대체.

Step 2: Min-Hashing (확률적 압축)

단일 min-hash: row의 랜덤 permutation \(\pi\) 선택. \(h_\pi(C)\) = (permuted 순서에서) column \(C\)가 1인 첫 번째 row의 index.

\[\Pr[h_\pi(C_1) = h_\pi(C_2)] = \text{sim}(C_1, C_2)\]

왜 성립하는가 (증명):

permuted 순서에서 \(C_1\) 또는 \(C_2\)가 1인 첫 번째 "active row"를 생각하자. active row들 중 정확히 \(|C_1 \cap C_2|\)개가 둘 다 1, 총 \(|C_1 \cup C_2|\)개. 첫 active row가 "둘 다 1" row일 확률 = Jaccard.

실제로는 permutation 안 씀

대신 hash 함수 사용: \(h_i(r) = (a_i \cdot r + b_i) \bmod p\). \(n\)개 선택 (예: \(n=100\)). Column \(C\)의 signature는 \([h_1, h_2, \ldots, h_n]\)의 벡터 (C=1인 row들에 대한 최솟값).

알고리즘 (streaming 구성)

# 초기화: sig[i][c] = ∞ (모든 i, c)
for each row r with value(c,r) = 1:
    for each hash i:
        sig[i][c] = min(sig[i][c], h_i(r))

\(C_1, C_2\)의 추정 Jaccard = signature가 일치하는 위치 비율.

L6 필수 암기

  • Jaccard 공식; signature = minhash 값의 벡터
  • 핵심 항등식: P(h(C1)=h(C2)) = Jaccard(C1,C2) — "first active row" 증명 필수
  • Signature 구성 알고리즘 (row 순차, min 업데이트)
  • 실제로는 permutation 대신 hash 함수
  • Signature 길이 = 선택한 hash 함수 수 (n)

L7: Finding Similar Items 2 — LSH Banding & Distance Measures

Intuition · 직관

Signature에서 Candidate Pair로

"We want to be hashed together only if we are likely similar."
유사할 가능성이 있을 때만 같이 해시되길 원함. Signature를 b개 band × r rows로 나눔 — 한 band이라도 같은 bucket이면 candidate.

핵심 트릭: 독립적인 hash table을 band마다 유지. 한 pair가 한 band이라도 같은 bucket에 들면 candidate. precision/recall 튜닝 가능.

Theory · 이론

LSH Banding

Signature 길이 \(n = b \cdot r\). \(b\)개 band로 나누고 각 band는 \(r\) rows.

  • 각 band마다 \(r\)개 값을 bucket에 해시 (band별 별도 hash table)
  • Candidate pair = 어느 한 band이라도 같은 bucket에 든 pair

S-Curve (Candidate Pair 확률)

\[P(\text{candidate pair}) = 1 - (1 - s^r)^b\]

\(s\) = 실제 Jaccard similarity.

  • \(s^r\) = 단일 band가 일치할 확률
  • \((1-s^r)^b\) = 모든 band가 불일치할 확률
  • \(1 - (1-s^r)^b\) = 적어도 한 band가 일치할 확률

Threshold 근사

\[t \approx \left(\frac{1}{b}\right)^{1/r}\]

S-curve가 \(t\) 근처에서 급격히 상승. 튜닝:

조절 (r×b=n 고정)t 변화S-curve결과
↑ r, ↓ bt 상승가파름precision↑, recall↓
↓ r, ↑ bt 하강완만recall↑, precision↓

외워둘 예: n=20, r=5, b=4. t ≈ (1/4)^(1/5) ≈ 0.76. s ≳ 0.76인 pair만 candidate.

False Positive / False Negative

  • False positive: 유사하지 않은 pair가 candidate → banding 후 직접 유사도 체크로 제거 가능
  • False negative: 유사한 pair를 놓침 → 추가 hash family 없이는 제거 불가

Distance Measure (거리 공리, 암기 필수)

\(d(x,y)\)가 metric이려면: (1) \(d \ge 0\), (2) \(d(x,y)=0 \iff x=y\), (3) symmetric, (4) triangle inequality.

Locality-Sensitive Hash Family

Family \(\mathcal{H}\)가 \((d_1, d_2, p_1, p_2)\)-sensitive 이려면, \(\mathcal{H}\)에서 랜덤 선택한 \(h\)에 대해:

  • \(d(x,y) \le d_1\)이면: \(\Pr[h(x)=h(y)] \ge p_1\)
  • \(d(x,y) \ge d_2\)이면: \(\Pr[h(x)=h(y)] \le p_2\)

의미 있으려면 \(d_1 < d_2\) AND \(p_1 > p_2\). AND/OR amplification = banding의 일반화.

[2021 Final] Q4 · LSH AND/OR Construction (8 points)
Given: LSH family F가 \((d_1, d_2, 0.6, 0.4)\)-sensitive. 즉 두 점이 가까우면 (\(d \le d_1\)) 충돌 확률 ≥ 0.6, 멀면 (\(d \ge d_2\)) ≤ 0.4.
(a) F의 member 2개로 AND-construction. \((d_1, d_2, w, x)\)의 w, x 계산.
(b) F의 member 2개로 OR-construction. \((d_1, d_2, y, z)\)의 y, z 계산.
(c) (b)의 OR 후 (a)의 AND로 cascading. fixed point p는?
정답:
(a) w = 0.36, x = 0.16
(b) y = 0.84, z = 0.64
(c) p = 1/(2-p)² 의 해, 약 0.38 (nontrivial fixedpoint)
풀이:
(a) AND-construction: 두 member 모두 충돌해야 함.
• 가까운 경우: \(w = 0.6^2 = 0.36\)
• 먼 경우: \(x = 0.4^2 = 0.16\)

(b) OR-construction: 적어도 하나 충돌하면 됨.
• 가까운 경우: \(y = 1 - (1-0.6)^2 = 1 - 0.16 = 0.84\)
• 먼 경우: \(z = 1 - (1-0.4)^2 = 1 - 0.36 = 0.64\)

(c) Cascading OR → AND: 입력 확률 p에 대해
• OR 후: \(p_1 = 1 - (1-p)^2 = 2p - p^2 = p(2-p)\)
• AND 후: \(p_2 = p_1^2 = p^2(2-p)^2\)
Fixed point: \(p = p^2(2-p)^2\). p ≠ 0이면 \(1 = p(2-p)^2\).
수치 해: p = 0.38에서 \(0.38 \times 1.62^2 \approx 0.997 \approx 1\) ⇒ p ≈ 0.38.

의미: cascading으로 두 확률 구간 모두 변함 — 이것이 LSH amplification의 핵심. 0.38 아래는 더 낮추고, 위는 더 높이는 효과.

L8: Clustering 1 — Hierarchical & k-means (계층적 + k-평균)

Intuition · 직관

두 가지 flavor

"Hierarchical builds a tree of merges — expressive but O(N³). Partitional picks k directly — fast but depends on k."
Hierarchical은 병합 트리를 만듦 — 표현력 있지만 O(N³). Partitional은 k를 직접 정함 — 빠르지만 k에 의존.

Clustering = 유사도 기반 비지도 그룹핑. 차원의 저주: 고차원에선 모든 점이 등거리로 보임.

Theory · 이론

Hierarchical Clustering (Agglomerative, 병합형)

  1. 시작: 각 점이 자기 cluster
  2. 반복: 가장 가까운 두 cluster 병합
  3. 원하는 cluster 수에 도달하면 중단 (또는 단일 cluster까지)

Cluster 간 거리 정의

Metric정의성질

Complete link최대 pairwise distancecompact한 둥근 cluster

k-means (Lloyd's Algorithm)

  1. 초기 k개 centroid 선택
  2. 각 점을 가장 가까운 centroid에 배정
  3. centroid를 배정된 점들의 평균으로 재계산
  4. 배정이 안 바뀔 때까지 2-3 반복

SSE \(\sum_{i}\sum_{x\in C_i}\|x-\mu_i\|^2\)의 local minimum에 수렴. Global optimum 보장 안 됨.

k-means++ (똑똑한 초기화)

목표: 초기 centroid를 퍼뜨려 나쁜 local min 회피.

  1. 첫 centroid \(c_1\)을 uniform random으로 선택
  2. 각 남은 점 \(p\)에 대해 \(D(p) = \min_{c_i} \|p - c_i\|\) 계산 (이미 선택된 centroid까지의 최소 거리)
  3. 다음 centroid를 확률 \(\frac{D(p)^2}{\sum_q D(q)^2}\)로 선택
  4. k개 centroid 고르고 standard k-means 실행
\[\Pr[\text{pick } p] = \frac{D(p)^2}{\sum_{q} D(q)^2}\]

정리: k-means++는 기대값 기준 O(log k)-approximation to optimal.

k 선택

  • Elbow method: WSS (within-cluster sum of squares) vs k 플롯. k 더 늘려도 개선 안 되는 "팔꿈치"점.
  • Silhouette: 각 점 \(i\)에 대해 \(s_i = (b_i - a_i) / \max(a_i, b_i)\). \(a_i\) = 자기 cluster 평균 거리, \(b_i\) = 가장 가까운 다음 cluster 평균 거리. 높을수록 좋음.
Exam Points · 시험 포인트

L8 필수 암기

  • k-means++ 확률 공식 (D²-weighted)
  • Lloyd's algorithm 단계; local min에만 수렴
  • Hierarchical: 4가지 link 타입 + 각각 언제 유리한지

L9: Clustering 2 — BFR & CURE (대용량 데이터용)

Intuition · 직관

데이터가 메모리에 안 들어갈 때

BFR: Gaussian 가정 하의 streaming k-means. cluster를 sufficient statistics (N, SUM, SUMSQ)로 요약.

CURE: 비구형 cluster를 여러 "representative point"로 표현.

Theory · 이론

BFR (Bradley-Fayyad-Reina)

가정: cluster는 axis-aligned Gaussian. 데이터를 청크 단위 로드.

3개 집합 유지:

Set무엇저장 형태
DS (Discard)cluster에 확실히 배정된 점들요약 통계 (N, SUM, SUMSQ)
CS (Compression)미배정 점 중 dense 서브클러스터요약 통계 (N, SUM, SUMSQ)
RS (Retained)아웃라이어, 아직 클러스터링 안 됨실제 점

cluster당 sufficient statistics (d 차원)

\[N: \text{count}, \quad \mathbf{SUM}_d: \sum x_d, \quad \mathbf{SUMSQ}_d: \sum x_d^2\] \[\text{centroid}_d = \frac{\text{SUM}_d}{N}, \quad \sigma_d^2 = \frac{\text{SUMSQ}_d}{N} - \left(\frac{\text{SUM}_d}{N}\right)^2\]

Mahalanobis 거리 (배정용)

\[\text{mahal}(x, c) = \sqrt{\sum_{d=1}^{D} \left(\frac{x_d - c_d}{\sigma_d}\right)^2}\]

규칙: 점 \(x\)의 mahal \(< \alpha \cdot \sqrt{D}\)이면 그 cluster에 배정 (흔한 threshold: α=2, 즉 mahal < 2√D). 아니면 RS로.

청크 하나 처리:

  1. 각 점: 어느 DS cluster의 Mahalanobis threshold 안이면 배정 + (N, SUM, SUMSQ) 업데이트. 아니면 RS로.
  2. RS의 점들 (이전 RS 포함) 에 k-means 적용; 작은 dense 서브클러스터 → CS, singleton은 RS 유지.
  3. 결합 variance가 작은 CS cluster끼리 merge.
  4. 마지막 청크: CS와 RS를 가장 가까운 DS cluster에 merge.

CURE (Clustering Using REpresentatives)

비구형 cluster 처리 (예: 길쭉한 두 모양).

  1. 메모리에 들어가는 양을 샘플링
  2. 샘플에 hierarchical clustering 적용
  3. 각 cluster에서 k개 representative point 선택 — cluster 내부에 퍼뜨림 (첫 번째: centroid에서 가장 먼 점, 그 다음: 이미 선택된 rep에서 가장 먼 점)
  4. Shrink: rep을 centroid 방향으로 α배만큼 이동 (흔히 α = 0.2):
    \[p'_i = p_i + \alpha(c - p_i) = (1 - \alpha) p_i + \alpha c\]
  5. 샘플에 안 뽑힌 점은 가장 가까운 shrunk rep의 cluster에 배정

Shrink 이유: 원본 rep이 outlier일 수 있음 → 안쪽으로 이동시켜 robustness. cluster가 주변 점까지 탐욕스럽게 먹지 않도록.

Novel Variant 경고: A-CURE (2025 출제)

A-CURE: shrinkage \(\alpha_i = 1/d_i\) (rep \(i\)에서 centroid까지 거리의 역수). 멀리 있는 rep은 덜 shrink, 가까운 rep은 더 shrink (극단적 역전). 비슷한 "시험 당일 정의되는" 변형 문제에 대비.

Exam Points · 시험 포인트

L9 필수 암기

  • BFR sufficient stats: N, SUM, SUMSQ 와 파생값 centroid/σ²
  • Mahalanobis distance 공식 (차원별 σ 사용)
  • DS/CS/RS 역할
  • CURE 5단계; shrink 공식 \(p' = (1-α)p + αc\)
  • 왜 shrink? outlier에 robust
T/F 함정:
- "BFR requires all data in memory" → FALSE (streaming 청크)
- "BFR assumes spherical clusters" → 부분 TRUE (axis-aligned Gaussian, 차원별 다른 σ 가능)
- "CURE uses one centroid per cluster" → FALSE (여러 rep)
- "Shrink moves toward nearest point" → FALSE (centroid 방향)

L10: Dimensionality Reduction 1 — SVD (특이값 분해)

Intuition · 직관

왜 SVD인가?

고차원 데이터는 보통 저차원 subspace에 있음. SVD는 Frobenius norm 기준 최적의 low-rank 근사를 찾음.

"Columns of U = concepts in user space. Columns of V = concepts in item space. Σ = strength of each concept."
U의 column = 사용자 공간의 concept. V의 column = 아이템 공간의 concept. Σ = 각 concept의 강도.
Theory · 이론

SVD 정의

\[M_{m \times n} = U_{m \times r} \, \Sigma_{r \times r} \, V^T_{r \times n}\]
  • \(r\) = rank of \(M\)
  • \(U\): orthonormal columns (left singular vectors)
  • \(V\): orthonormal columns (right singular vectors)
  • \(\Sigma\): diagonal, 양수 내림차순 (singular value \(\sigma_1 \ge \sigma_2 \ge \cdots \ge 0\))

해석

\(M = \sum_{i=1}^{r} \sigma_i \, u_i \, v_i^T\) — 중요도 순 rank-1 matrix의 합.

Concept: 각 rank-1 항이 잠재 factor. 예: 영화 평점 → factor 1 = SF 좋아함, factor 2 = 로맨스 좋아함.

최적 Low-Rank 근사 (Eckart-Young)

상위 \(k\)개 singular value만 유지: \(M_k = \sum_{i=1}^{k} \sigma_i u_i v_i^T\). 그러면:

\[\|M - M_k\|_F^2 = \sum_{i=k+1}^{r} \sigma_i^2\] \[M_k = \arg\min_{\text{rank}(B)=k} \|M - B\|_F\]

Energy

\[\text{Total energy} = \sum_{i=1}^{r} \sigma_i^2 = \|M\|_F^2\] \[\text{Top-k 유지 에너지 비율} = \frac{\sum_{i=1}^{k} \sigma_i^2}{\sum_{i=1}^{r} \sigma_i^2}\]

경험칙: ≥ 90% energy 유지하도록 \(k\) 선택.

Eigendecomposition과의 관계

\(M^T M = V \Sigma^2 V^T\). V의 column = \(M^T M\)의 eigenvector. Eigenvalue = \(\sigma_i^2\).

계산 (2D worked example)

\(M\) (m×n) 주어지면, \(M^T M\) (n×n) 계산. Eigendecompose: \(M^T M = V \Lambda V^T\), \(\lambda_i = \sigma_i^2\). 그 후 \(U = M V \Sigma^{-1}\).

SVD로 쿼리·추천

user row \(q\)를 concept space로 projection: \(q_{\text{concept}} = q V\). \(U\Sigma\) 공간에서 가장 가까운 user 찾기. 해당 concept 차원에서 값 높은 아이템 추천.

Exam Points · 시험 포인트

L10 필수 암기

  • SVD factorization shape: U (m×r), Σ (r×r), V^T (r×n)
  • \(M_k\)가 Frobenius norm 기준 최적, 오차 = \(\sqrt{\sum_{i>k}\sigma_i^2}\)
  • Energy 공식 (\(\sigma^2\) 합) 과 retained energy 비율
  • \(V\) = \(M^T M\)의 eigenvector, eigenvalue = \(\sigma_i^2\)
  • U와 V는 orthonormal column (row는 일반적으로 아님)
T/F 함정:
- "SVD is unique" → FALSE (singular vector 부호 뒤집기; 같은 σ 부분공간에서 회전 가능)
- "Σ is always square" → compact SVD에선 TRUE (r×r); full SVD면 U(m×m), V(n×n), Σ(m×n)로 zero padding
- "SVD minimizes L1 norm" → FALSE (Frobenius/L2)

L11: Dimensionality Reduction 2 — CUR

Intuition · 직관

왜 CUR인가?

SVD의 문제: (1) \(M\)이 sparse해도 \(U, V\)가 dense → 저장 공간 폭발. (2) \(U, V\)의 column이 row/column의 linear combination → 해석 어려움.

CUR: \(M \approx C \cdot U \cdot R\)로 근사. \(C\) = \(M\)의 실제 column, \(R\) = 실제 row, \(U\) = 작은 중간 행렬. sparsity와 해석성 유지.

Theory · 이론

CUR Decomposition

\[M_{m \times n} \approx C_{m \times r} \cdot U_{r \times r} \cdot R_{r \times n}\]

Sampling 확률

column \(j\)를 선택할 확률 (C 구성용):

\[p_j = \frac{\|m_{\cdot j}\|^2}{\|M\|_F^2} = \frac{\sum_i m_{ij}^2}{\sum_{i,k} m_{ik}^2}\]

row에도 동일 방식 (R 구성용).

Column 스케일링 (매우 중요)

column \(j\)를 C에 추가할 때 \(1/\sqrt{r \cdot p_j}\)로 스케일링 — expectation이 원본과 맞도록.

중간 행렬 U

  1. \(W\) = 선택된 row와 column의 교집합 (\(r \times r\))
  2. SVD: \(W = X \Sigma Y^T\)
  3. \(U = Y \Sigma^+ X^T\) (\(\Sigma^+\) = Moore-Penrose pseudoinverse, 0이 아닌 항만 역수)

보장

\(r = O(k \log k / \epsilon^2)\) 샘플이면 expectation 기준 \(\|M - CUR\|_F \le (1+\epsilon)\|M - M_k\|_F\). SVD rank-k의 (1+ε)-approximation이지만 해석성 유지.

SVD vs CUR 비교

SVDCUR
정확도최적 (rank-k)(1+ε) of optimal
Factor sparsity보통 denseSparse (실제 row/col 사용)
해석성Linear combo — 추상실제 row/col — 구체
중복 column깔끔하게 처리재샘플링 가능 (oversampling)
Exam Points · 시험 포인트

L11 필수 암기

  • Column 확률: \(p_j = \|m_{\cdot j}\|^2 / \|M\|_F^2\)
  • 샘플 후 스케일링: \(1/\sqrt{r p_j}\)
  • U는 W=C∩R의 pseudoinverse로 구성
  • CUR은 SVD rank-k의 (1+ε)-approximation
  • 중복 column 이슈: oversampling으로 처리
T/F 함정:
- "CUR gives exact equality to M" → FALSE (approximation)
- "CUR picks columns uniformly at random" → FALSE (column squared norm에 비례)
- "U in CUR is diagonal like Σ" → FALSE (일반 r×r)

L12: Recommender Systems 1 — Content & Collaborative Filtering

Intuition · 직관

두 가지 접근

Content-based: 아이템 특성 기반 — user가 좋아했던 것과 비슷한 아이템 추천.

Collaborative Filtering (CF): 상호작용만 기반 — 당신과 "비슷한" user들이 좋아한 아이템 추천. item feature 불필요.

Theory · 이론

Utility Matrix

\(R_{ui}\) = user \(u\)가 item \(i\)에 준 rating (또는 implicit feedback). 대부분 매우 sparse (거의 비어있음).

Content-based

  1. item profile = feature 벡터 (예: 텍스트 TF-IDF, 장르 one-hot)
  2. user profile = 유저가 좋아한 아이템들의 가중 평균
  3. 예측: score = cosine(user profile, item profile)

장점: 새 아이템 cold-start OK, 설명 가능. 단점: feature 필요, over-specialization (serendipity 없음).

Collaborative Filtering: User-User

User similarity (Pearson correlation)

\(S_{xy}\) = x와 y가 둘 다 평가한 아이템. \(\bar r_x, \bar r_y\) = 유저 평균 (\(S_{xy}\) 또는 전체 평가).

\[\text{sim}(x, y) = \frac{\sum_{i \in S_{xy}} (r_{xi} - \bar r_x)(r_{yi} - \bar r_y)}{\sqrt{\sum (r_{xi} - \bar r_x)^2}\sqrt{\sum (r_{yi} - \bar r_y)^2}}\]

평균 빼는 이유: 유저마다 rating 스케일 다름 (harsh vs lenient). 중심화 후 cosine.

예측 (k-NN weighted)

\[\hat r_{xi} = \bar r_x + \frac{\sum_{y \in N(x;i)} \text{sim}(x,y) \cdot (r_{yi} - \bar r_y)}{\sum_{y \in N(x;i)} |\text{sim}(x,y)|}\]

Item-Item CF (보통 더 좋음)

같은 수학이지만 item 기준. item i와 비슷하면서 x가 평가한 아이템을 찾아 weighted average.

\[\hat r_{xi} = \bar r_x + \frac{\sum_{j \in N(i;x)} \text{sim}(i,j) \cdot (r_{xj} - \bar r_x)}{\sum |\text{sim}(i,j)|}\]

왜 item-item이 user-user보다 나은가: item 유사도가 더 안정적 (아이템 특성은 일관됨; 유저는 기분 바뀜).

평가 (Evaluation)

  • rating을 train/test로 분리; test rating 예측
  • RMSE: \(\sqrt{\frac{1}{|T|}\sum (r - \hat r)^2}\)
  • MAE: 평균 절대 오차
  • Top-k accuracy: 상위 예측 리스트에 test item이 포함되는지?

Cold Start

  • 새 user / 새 item — 이력 없음
  • Content-based는 새 item 처리 가능 (feature 있음)
  • CF 실패 — hybrid 또는 content fallback 필요
Exam Points · 시험 포인트

L12 필수 암기

  • Pearson correlation 공식 — 유저 평균으로 중심화
  • User-user vs item-item CF 예측 공식
  • Content-based vs CF 장단점
  • RMSE 정의
  • Cold start trade-off
T/F 함정:
- "Content-based handles cold-start for both users and items" → FALSE (아이템만)
- "User-user and item-item give identical predictions" → FALSE
- "Pearson is computed over all items" → FALSE (공통 평가된 아이템만)

L13: Recommender Systems 2 — Latent Factors, SGD, BPR

Intuition · 직관

Neighborhood에서 Latent Factor로

CF neighborhood 방식은 local + memory-based. Latent-factor 방식은 user/item의 k차원 embedding을 학습: \(r_{ui} \approx u_x \cdot v_i\). 훨씬 scalable하고 보통 더 정확.

Theory · 이론

UV Decomposition

\[R \approx U V^T, \quad U \in \mathbb{R}^{m \times k}, \; V \in \mathbb{R}^{n \times k}\] \[\hat r_{ui} = u_x^\top v_i\]

목표: 관측된 entry에 대해서만 SSE 최소화:

\[\text{SSE} = \sum_{(x,i) \in \text{obs}} (r_{xi} - u_x^\top v_i)^2 + \lambda(\|U\|_F^2 + \|V\|_F^2)\]

UV vs SVD — 핵심 차이

UVSVD
직교성 제약 없음U, V orthonormal
blank entry 무시 (sparse-aware)모든 entry 처리 (0 또는 imputation)
SGD / ALS로 해결linear algebra 정확해
k는 overfit 회피로 선택k는 energy 유지로 선택

Stochastic Gradient Descent (SGD) Update

rating \(r\)을 가진 관측된 \((x, i)\) 마다:

\[\varepsilon = 2(r - u_x^\top v_i)\] \[u_x \leftarrow u_x + \eta(\varepsilon \, v_i - \lambda \, u_x)\] \[v_i \leftarrow v_i + \eta(\varepsilon \, u_x - \lambda \, v_i)\]

\(\eta\) = learning rate, \(\lambda\) = regularization.

Alternating Least Squares (ALS)

  1. \(V\) 고정, \(U\) 해결: 행 단위 ridge regression (closed form)
  2. \(U\) 고정, \(V\) 해결: 동일
  3. 반복

row/column 단위 병렬화 가능. implicit feedback (전체 행렬 dense)에 특히 유리.

Bias Term

\[\hat r_{xi} = \mu + b_x + b_i + u_x^\top v_i\]

\(\mu\) = global 평균, \(b_x, b_i\) = user/item bias. "user가 harsh" / "영화가 전반적으로 사랑받음"을 latent factor 낭비 없이 포착.

Implicit Feedback & BPR

명시적 rating 없이 click/view/purchase만 있을 때. 관측된 pair를 "positive", 미관측을 "negative but uncertain"으로 취급.

BPR (Bayesian Personalized Ranking): user \(x\)가 관측된 item \(i\)를 미관측 \(j\)보다 선호할 가능성을 최대화.

\[J_\text{BPR} = \sum_{(x,i,j)} -\log \sigma(u_x^\top v_i - u_x^\top v_j) + \lambda \|\theta\|^2\]

Pairwise loss: \(r_{xi}\)는 관측되고 \(r_{xj}\)는 미관측일 때 \(i\)를 \(j\) 위로 랭킹. missing-not-at-random 상황 올바르게 처리.

[2025 Final] Q1(c) T/F · UV vs SVD with blanks
"In UV decomposition for recommender systems, we can safely apply standard SVD directly to the sparse rating matrix by treating all missing ratings as zeros."
정답: FALSE
풀이: 핵심 함정. SVD는 blank를 '0점 평가'로 간주하여 factorize → 평가하지 않은 것과 '싫어함(0점)'을 섞어버림 → 왜곡된 결과. UV는 관측된 rating만 SSE에 포함 → 올바른 접근. SVD를 blank→0 imputation으로 쓰는 건 "safely"가 아님.
[2025 Final] Q1(d) T/F · L2 regularization effect
"L2 regularization in latent factor models tends to pull user and item vectors toward the origin, so only strong factors remain large."
정답: TRUE
풀이: 손실 \(\text{SSE} + \lambda(\|U\|_F^2 + \|V\|_F^2)\). gradient에 \(-\lambda u\) 항 → 매 step u가 0 방향으로 수축(shrinkage). 데이터가 충분히 뒷받침하는 factor만 크게 남고 noise factor는 억제됨.
[2022 Final] Q1(a) T/F · UV global minimum
"The UV-decomposition algorithm can sometimes return a global minimum solution."
정답(추정): TRUE
풀이: "sometimes"가 핵심. SGD/ALS는 non-convex이지만 (1) 좋은 초기화 + 운으로 global min 도달 가능, (2) rank=1 케이스는 closed-form global min. 절대 도달 못 한다고 단정할 수 없음.

📋 핵심 공식 카드 (Formula Quick-Reference)

L1 — TF-IDF / Hash / Power Law

  • \(\text{TF-IDF}(i,j) = \text{TF}(i,j) \cdot \log(N/n_i)\)
  • Hash: \(h(k) = k \bmod B\) maps key → bucket
  • Power law: \(y = c x^a \iff \log y = a \log x + \log c\)

L2 — MapReduce

  • Map: \((k,v) \to \{(k_i,v_i)\}\); Reduce: \((k,[v_1,\ldots]) \to (k,v)\)
  • MV multiplication: map \(((i,j), m_{ij}) \to (i, m_{ij} v_j)\)
  • Combiner condition: commutative AND associative

L4-L5 — Itemsets

  • Support(I) = # baskets with I; Confidence(I→J) = supp(I∪J)/supp(I); Interest = Conf − P(J)
  • Triangular memory: \(4 \cdot n(n-1)/2\); Triples: \(12 p \cdot n(n-1)/2\); triples better if p < 1/3
  • A-Priori: \(C_k \to L_k \to C_{k+1}\) pipeline
  • PCY: Pass1 uses bitmap (1/32 space). Must use triples.
  • SON correctness: pigeonhole → if \(\text{supp}(S) \ge s\), some chunk has \(\text{supp}(S) \ge ps\)

L6-L7 — LSH

  • Jaccard: \(J(A,B) = |A \cap B|/|A \cup B|\)
  • MinHash identity: \(\Pr[h(C_1)=h(C_2)] = J(C_1,C_2)\)
  • S-curve: \(P = 1 - (1-s^r)^b\); Threshold: \(t \approx (1/b)^{1/r}\)
  • r↑ (b↓): steeper curve, precision↑ recall↓

L8-L9 — Clustering

  • k-means++ prob: \(D(p)^2 / \sum D(q)^2\)
  • BFR stats: \(N, \text{SUM}_d, \text{SUMSQ}_d\); centroid=SUM/N, \(\sigma^2 = \text{SUMSQ}/N - (\text{SUM}/N)^2\)
  • Mahalanobis: \(\sqrt{\sum_d ((x_d - c_d)/\sigma_d)^2}\), threshold \(2\sqrt{D}\)
  • CURE shrink: \(p' = (1-\alpha) p + \alpha c\)
  • Silhouette: \((b - a)/\max(a, b)\)

L10-L11 — SVD/CUR

  • SVD: \(M = U \Sigma V^T\); Frobenius energy \(= \sum \sigma_i^2\); retention \(= \sum_{i\le k}\sigma_i^2 / \sum \sigma_i^2\)
  • Rank-k error: \(\|M-M_k\|_F^2 = \sum_{i>k}\sigma_i^2\)
  • CUR column prob: \(p_j = \|m_j\|^2 / \|M\|_F^2\); scale by \(1/\sqrt{r p_j}\)
  • U = Y Σ⁺ X^T with \(W = X\Sigma Y^T\) SVD of intersection

L12-L13 — RecSys

  • Pearson: \(\frac{\sum(r_{xi}-\bar r_x)(r_{yi}-\bar r_y)}{\sqrt{\sum(r_{xi}-\bar r_x)^2}\sqrt{\sum(r_{yi}-\bar r_y)^2}}\)
  • Item-item prediction: \(\hat r_{xi} = \bar r_x + \frac{\sum_{j \in N(i;x)} \text{sim}(i,j)(r_{xj}-\bar r_x)}{\sum |\text{sim}(i,j)|}\)
  • UV SSE: \(\sum_{(x,i) \in \text{obs}}(r_{xi} - u_x^T v_i)^2 + \lambda(\|U\|_F^2 + \|V\|_F^2)\)
  • SGD: \(\varepsilon = 2(r - u^T v)\); \(u \leftarrow u + \eta(\varepsilon v - \lambda u)\); \(v \leftarrow v + \eta(\varepsilon u - \lambda v)\)
  • Bias: \(\hat r = \mu + b_x + b_i + u^T v\)
  • BPR loss: \(\sum -\log \sigma(u^T v_i - u^T v_j)\)
  • RMSE: \(\sqrt{\frac{1}{|T|}\sum(r - \hat r)^2}\)

  • ☐ A-Priori의 support threshold s를 올리면? — 후보 수 급감, 패스 속도↑, recall↓
  • ☐ PCY에서 bucket 수 B를 늘리면? — 각 bucket 평균 count↓ → bitmap 희소해짐 → 후보 pair↓
  • ☐ Toivonen에서 ps → 0.9ps로 낮추면? — negative border 커짐, fail 확률↓
  • ☐ LSH에서 r 증가 (n 고정)? — threshold↑, precision↑, recall↓
  • ☐ k-means++에서 D²를 D로 바꾸면? — far outlier 영향↓, 중간 거리 점들 사이 분산
  • ☐ BFR의 Mahalanobis threshold를 3√D로 늘리면? — DS에 더 포함, CS/RS 수↓
  • ☐ CURE shrink α가 1이면? — 모든 rep이 centroid에 수렴 → k-means와 동일해짐
  • ☐ SVD의 k를 rank 미만으로 줄이면? — Frobenius 오차 = Σ_{i>k} σ_i²만큼 발생
  • ☐ UV의 λ를 0으로 하면? — 정규화 없음 → overfit 가능
